[[2509.21241] Explaining Fine Tuned LLMs via Counterfactuals A Knowledge Graph Driven Framework](https://arxiv.org/abs/2509.21241)

通过反事实解释微调的 LLM知识图谱驱动的框架

## 摘要

低秩自适应 （LoRA） 的广泛采用使大型语言模型 （LLM） 能够以极高的效率获取特定领域的知识。然而，了解这种微调机制如何改变模型的结构推理和语义行为仍然是一个悬而未决的挑战。这项工作引入了一个新颖的框架，通过基于知识图谱的反事实来解释微调的法学硕士。具体来说，我们构建了生物信息学工具中特定领域的异构知识图谱 BioToolKG，并设计了一个基于反事实的微调 LLMs 解释器 （CFFTLLMExplainer），该解释器在图节点和边缘上学习软掩码，以产生最小的结构扰动，从而引起最大的语义发散。我们的方法共同优化了结构稀疏性和语义发散，同时强制执行可解释性，保留了熵正则化和边缘平滑度等约束。我们将该框架应用于微调的基于 LLaMA 的 LLM，并揭示反事实掩蔽暴露了模型的结构依赖性，并与 LoRA 引起的参数偏移保持一致。这项工作为微调法学硕士的内部机制提供了新的见解，并强调了反事实图作为可解释人工智能的潜在工具。

# 论文速读

## Q1: 这篇论文试图解决什么问题？

该研究面向“微调后的大语言模型（LLM）如何获得领域知识”这一黑箱过程，提出以下核心问题：

- 低秩适配（LoRA）仅更新极少量参数即可显著改变模型行为，其内部究竟编码了何种“结构敏感”模式？
- 当输入为结构化知识图谱时，模型是否依赖特定节点/边进行推理？这些依赖能否与 LoRA 引入的参数偏移建立对应关系？
- 现有解释方法（注意力、prompt 扰动、SHAP 等）面对生成式 LLM 时存在语义度量困难、无法揭示内部机制等局限，如何设计一种面向图结构、可解释且与微调过程挂钩的新解释框架？

论文通过构建领域知识图谱 BioToolKG，并设计基于反事实的子图掩码框架 CFFTLLMExplainer，将上述问题形式化为“最小结构扰动 → 最大语义漂移”的多目标优化任务，从而揭示 LoRA 微调后模型对图结构组件的敏感性与偏好，实现对微调机制的结构级解释。

## Q2: 有哪些相关研究？

论文在第 2 节系统回顾了与“LLM 可解释性”及“LoRA 微调”直接相关的研究，可归纳为以下四条主线：

1. 参数高效微调
   
   - LoRA 系列：Hu et al. 2022 提出低秩适配 W=W0+αAB，仅训练 A,B 两个小矩阵即可注入领域知识。
   - 后续变体（未列出）包括 AdaLoRA、DoRA、QLoRA 等，均聚焦效率与性能，但未解释适配后的行为变化。

2. 经典 XAI 方法在 LLM 上的迁移与局限
   
   - 特征扰动类：LIME、SHAP、DiCE 通过输入扰动衡量特征贡献，原本面向分类器。
   - 梯度/积分类：Integrated Gradients、Sequential Integrated Gradients 直接对 token 级嵌入求梯度，难以处理自由文本输出。
   - 注意力可视化：Vaswani et al. 2017 提出自注意力，但 Jain & Wallace 2019、Wiegreffe & Pinter 2019 等指出“注意力 ≠ 解释”，深层多头注意力与推理路径未必一致。
   - Token 级 SHAP：TokenSHAP（Horovicz & Goldshmidt 2024）用蒙特卡洛估计 Shapley 值，仍受限于生成任务的高维输出空间。

3. Prompt-based 探测与自解释
   
   - 链式思维（CoT）、least-to-most prompting 等通过修改提示观察输出变化，但 Turpin et al. 2023、Huang et al. 2023 证明 LLM 可能给出“忠实度低”的自然语言理由，与内部决策过程不一致。
   - Webson & Pavlick 2022 进一步指出模型对 prompt 的语义理解并不稳健，解释结果易受措辞影响。

4. 结构/概念瓶颈解释
   
   - CB-LLM（Sun et al. 2024）在模型内部插入“概念瓶颈层”，强制输出可解释概念权重，但需要重新训练主模型，不适用于已微调的 black-box LLM。
   - 图神经网络解释器（GNNExplainer、PGExplainer 等）提供子图掩码思路，但对象为图分类任务，未涉及文本生成或 LoRA 适配场景。

综上，已有工作要么聚焦分类器解释，要么仅分析注意力/提示，缺乏“结构化知识图谱 + 参数高效微调”双重设定下的机制揭示。本文首次将反事实子图掩码与 LoRA 低秩子空间分析结合，填补了这一空白。

## Q3: 论文如何解决这个问题？

论文将“解释 LoRA 微调后 LLM 如何依赖知识图谱结构”这一难题转化为**可微反事实掩码学习**问题，通过**结构稀疏扰动**诱导**最大语义漂移**，进而逆向揭示模型内部敏感模式。具体解法分为三步：

### 1. 构造领域知识图谱 BioToolKG

- 8 类实体、14 种关系，聚焦生物信息学工具链的“输入-输出”数据流。
- 将图实例化为**路径寻找任务**：给定数据库节点 → 按输入/输出约束生成合法工具流水线。
- 用指令式 prompt 把图转化为文本，使 LLM 可直接消费，同时保留结构语义。

### 2. 提出反事实解释框架 CFFTLLMExplainer

把“解释”定义为**最优反事实子图** Gc 的生成问题：

G∗c=argminGcd(G,Gc)s.t.fft(T(Gc))≠fft(T(G))

其中 T(⋅) 为图→文本模板，fft 是 LoRA 微调后的模型。为实现端到端学习，引入**可微软掩码**：

- 节点掩码 mv∈[0,1]、边掩码 me∈[0,1]，用 Gumbel-Sigmoid 重参数化： 
  
  z~=σ((logα+G)/τ),G∼Gumbel

- 掩码后子图 Gc 与原图 G 同时喂入 LLM，用 TF-IDF 向量计算**语义漂移** Lsemantic。

### 3. 设计多目标损失，联合优化“稀疏+漂移+可解释”

总损失：

- Lstructure 按节点/边类型加权，鼓励删除“低成本”元素。
- Lsemantic 可再被 prompt 相似度动态加权，防止删光高相关节点。
- Lsmooth 令边掩码与两端节点掩码保持线性一致，避免孤立边，保证生物合理性。

### 4. 解释分析：对齐“掩码-注意力-LoRA 偏移”

得到最优 Gc 后，从三视角验证解释有效性：

| 视角         | 指标                           | 发现                                                       |
| ---------- | ---------------------------- | -------------------------------------------------------- |
| 语义漂移       | Jaccard、Edit Distance、Cosine | 仅 10% 工具重叠，漂移显著高于随机或高注意力删点。                              |
| 注意力对齐      | 平均注意力权重                      | 被删节点注意力极低（<0.002），说明**注意力≠重要性**。                         |
| Adapter 偏移 | ∥Δ∥2=∥BA⋅e∥2                 | 被删节点嵌入偏移最大（Scallop 0.0102），与掩码选择高度一致，证实 LoRA 子空间编码了结构偏好。 |

### 5. 与基线对比

- 随机删点/边：结构破坏大，但语义变化小（Jaccard 仍 0.4–0.6）。
- 高注意力删点：漂移略高，但破坏生物流程连贯性，解释性差。
- CFFTLLMExplainer：在**保持图谱合法路径**的前提下实现最大语义偏离，验证其“最小结构扰动→最大行为改变”的目标达成。

综上，论文通过**可微反事实掩码+多目标优化**，首次把“LoRA 学到的低秩子空间”与“知识图谱结构依赖”显式挂钩，为理解参数高效微调后的 LLM 提供了一种**结构级、可验证、与参数偏移对齐**的新解释范式。

## Q4: 论文做了哪些实验？

论文围绕“反事实掩码能否揭示 LoRA 微调后 LLM 的结构依赖”这一核心假设，设计并执行了两组共 7 种扰动策略的实验。所有实验均基于同一条真实转录组拼接流水线（图 2）及同一对模型（baseline vs LoRA 微调 8B 模型）。结果从语义漂移、结构合法性、注意力对齐、Adapter 偏移四个维度量化对比。

### 实验 1：反事实解释验证（Section 4.1）

目的：验证 CFFTLLMExplainer 学到的最小掩码是否恰好击中模型敏感结构，并能否在 LoRA 参数层面找到对应信号。  
流程：

1. 用完整图 G 分别喂给 baseline 与 ft 模型 → 得到 fbase(G)、fft(G)。
2. 用 CFFTLLMExplainer 优化出反事实子图 Gc（仅保留≈40 % 节点/边）。
3. 将 Gc 喂给 ft 模型 → 得到 fft(Gc)。
4. 对比三套输出，计算：
   - 语义漂移：Jaccard、Edit Distance、Path Overlap、Cosine Dissimilarity
   - 注意力对齐：提取工具 token 的平均注意力权重
   - Adapter 偏移：对每条工具 token 计算 Δ=BA·e 的 ℓ2 范数

关键结果（表 1）：

- Jaccard 仅 0.10，Cosine 距离 0.54，表明掩码成功诱导显著语义变化。
- 被删节点注意力权重均 <0.002，说明注意力与“重要性”错位。
- Scallop、Gffcompare、Samtools 的 ∥Δ∥ 最大，与掩码选择完全一致，证实 LoRA 子空间编码了这些工具的结构偏好。

### 实验 2：基线扰动对照（Section 4.2）

目的：回答“如果不按反事实目标，仅随机或按注意力删点/边，能否产生同等语义漂移？”  
共 7 种扰动策略，每种都在 baseline 与 ft 模型上分别测试，保持删除规模与 Gc 完全一致：

| 策略                      | 说明                 |
| ----------------------- | ------------------ |
| RandomNodeMask          | 随机删同数量节点（类型不限）     |
| RandomEdgeMask          | 随机删同数量边            |
| RandomNodeEdgeMask      | 随机混合删节点+边          |
| RandomNodeMaskAlign     | 随机删节点，但类型分布与 Gc 对齐 |
| RandomNodeEdgeMaskAlign | 同上，节点+边均对齐         |
| HigherAttention         | 删除注意力权重最高的同类型节点    |
| LowerAttention          | 删除注意力权重最低的同类型节点    |

评价指标：

- Jaccard（语义重叠）
- Edit Distance normalized（结构偏差）
- Overlap（生成 toolchain 最长公共前缀比例）

结果（图 7 + 表 3）：

- 随机类策略 Jaccard 普遍 >0.4，甚至 >0.6，语义变化微弱；且常产生生物无效断链。
- HigherAttention 策略 Jaccard≈0.14，优于随机，但仍远高于 Gc 的 0.10；且结构断裂明显。
- CFFTLLMExplainer(Gc) 同时取得**最低 Jaccard、最高 Edit、最低 Overlap**，且保留的流水线在生物上仍然合法，证明其“小结构扰动-大语义改变”优势并非偶然。

### 附加分析

- 训练曲线（图 3）：语义损失单调上升，稀疏损失单调下降，总损失平稳收敛。
- 掩码可视化（图 4）：深色=高保留概率，直观显示被丢弃的正是工具链核心组件。
- 三信号热图（图 6）：节点掩码分数、注意力分数、Adapter 偏移三者并排，肉眼可见“掩码高亮 ≠ 注意力高亮”，但高度吻合 Adapter 偏移大小，强化“结构敏感信号藏在 LoRA 子空间”这一结论。

综上，实验从“能否诱导漂移—是否结构最小—是否对齐内部参数”三个层次系统验证：CFFTLLMExplainer 提供的反事实掩码不仅能以最小结构变动最大化改变模型输出，而且其选择与 LoRA 低秩偏移一一对应，显著优于随机或注意力启发式扰动。

## Q5: 有什么可以进一步探索的点？

作者在第 6 节已给出三条未来方向，结合实验未触及的空白，可进一步探索的关键点归纳如下：

1. **提示级反事实**  
   当前掩码在“图→文本”之后执行，下一步可直接对**自然语言提示**学习 token 级掩码，验证结构敏感度是否随措辞变化而漂移，并建立统一的“图-文本”双空间反事实框架。

2. **人类反馈约束**  
   将生物学家先验（工具依赖表、必含路径、禁用组合）编码为**硬约束或即时奖励**，实现**人机协同反事实生成**，提升解释的可信度和实用性。

3. **跨域与多任务迁移**  
   把 BioToolKG 换成法律判例图、化学合成路径或软件 API 调用图，检验 CFFTLLMExplainer 是否**无需修改超参**即可揭示不同领域 LoRA 适配后的结构偏好，验证框架通用性。

4. **多模态与动态图**  
   引入**实验数据表、图像或时序日志**作为节点属性，研究当输入为**动态异构图**（节点/边随时间增减）时，反事实掩码能否捕捉**时序依赖**与**模态交互**。

5. **参数效率与因果干预**  
   结合因果推断，对 LoRA 子空间 A,B 做**do-intervention**（如删除某列/行），观测下游工具链概率变化，量化**因果强度**，并与掩码重要性排序做一致性检验。

6. **可解释性评价基准**  
   目前缺乏金标准。可邀请领域专家标注“哪些节点/边对决策至关重要”，构建**人工重要性标签**，用以计算 AUPRC 或 Ranking-Kendall，系统评估不同解释方法的忠实度。

7. **计算加速与离散化**  
   现采用 Gumbel-Sigmoid+连续松弛，训练步数多。可尝试**Straight-Through Estimator** 或 **0/1 整数规划近似**，在保持梯度可传的同时直接输出离散掩码，缩短收敛时间。

8. **对抗鲁棒性诊断**  
   利用反事实掩码生成**最小图级对抗触发器**，检测 LoRA 模型是否比 baseline 更易受特定结构扰动影响，为**安全部署**提供预警。

9. **多 LoRA 模块联合分析**  
   当前仅探测 query-proj 模块。可对**所有 LoRA 层**（k_proj, v_proj, o_proj, FFN）同时学习掩码，研究不同模块对结构组件的**分工敏感度**，构建**模块级重要性热图**。

10. **自监督预训练掩码**  
    在**无下游任务标签**阶段，先以“掩码→预测被删节点/边类型”为代理任务预训练掩码生成器，再迁移到解释目标，看是否能**零样本**产生高忠实度反事实。

这些方向分别从输入模态、约束形式、评价指标、因果深度、计算效率与安全维度出发，可系统拓展 CFFTLLMExplainer 的适用范围与理论深度。

## Q6: 总结一下论文的主要内容

- **问题**：LoRA 微调仅更新极少量参数即可让大语言模型获得领域知识，但其“结构依赖”与内部机制仍属黑箱；现有解释方法难以处理生成式输出且与 LoRA 参数偏移脱节。

- **思路**：把“解释”转化为**最小图结构扰动 → 最大语义漂移**的反事实优化——通过可微软掩码删除知识图谱的节点/边，观察微调模型输出变化，从而逆向揭示敏感结构。

- **方法**：
  
  1. 构建生物信息学异构知识图谱 BioToolKG，定义工具链路径寻找任务。
  
  2. 提出 CFFTLLMExplainer：为每节点/边学习掩码 m∈[0,1]，用 Gumbel-Sigmoid 保证可微，联合优化 
     
     Ltotal=Lsparse+αLsemantic+βLentropy+γLpreserve+δLhard+εLsmooth
     
      以平衡稀疏、漂移、二值化与生物合理性。
  
  3. 对齐三重视角：掩码重要性、token 注意力、LoRA 低秩偏移 ∥BAe∥2，验证结构敏感信号是否嵌入适配子空间。

- **实验**：
  
  - 反事实验证：完整图 vs 掩码图 → Jaccard 从 1 降至 0.10，Cosine 距离 0.54；被删节点注意力极低却对应最大 Adapter 偏移，表明注意力≠重要性，而 LoRA 子空间显式编码这些工具。
  - 基线对照：随机删、高注意力删均无法同时实现“小结构破坏+大语义变化”，且常产生生物无效断链，证明反事实掩码更具解释力与领域合法性。

- **结论**：首次提供**结构级、可验证、与参数偏移对齐**的 LoRA 微调解释范式，强调“图结构敏感模式”可通过最小反事实扰动被精准挖掘并可视化。
