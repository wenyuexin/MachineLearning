## 一、论文整体思路

### 1.1 研究背景

深度学习模型在各类任务上取得了卓越性能，但其**校准性能（Calibration）**往往被忽视。准确率高不代表模型可靠——如果模型在错误预测时仍给出高置信度，可能导致严重后果。本综述系统梳理了深度学习领域的校准研究进展，涵盖度量方法、校准技术、以及不同任务中的应用。

### 1.2 综述范围

本综述全面覆盖深度学习校准的各个方面：
1. **校准度量**：ECE、Brier Score、NLL等经典度量及其变体
2. **校准方法**：训练时方法与后处理方法
3. **任务应用**：分类、检测、分割等不同任务的校准挑战
4. **前沿方向**：联邦学习、自监督学习、大模型等新兴场景

### 1.3 主要贡献

1. **全面系统**：覆盖校准研究的各个维度
2. **任务导向**：详细分析不同深度学习任务的校准特点
3. **方法对比**：系统对比各类校准方法的优劣
4. **前沿追踪**：涵盖2023年之前的最新进展

## 二、校准度量体系

### 2.1 经典度量

#### 2.1.1 ECE（期望校准误差）

$$
ECE = \sum_{m=1}^{M} \frac{|B_m|}{N} \cdot |acc(B_m) - conf(B_m)|
$$

**特点**：
- 最广泛使用的校准度量
- 基于分桶的近似估计
- 直观反映置信度-准确率差距

#### 2.1.2 Brier Score

$$
BS = \frac{1}{N} \sum_{i=1}^{N} \sum_{c=1}^{C} (p_{i,c} - y_{i,c})^2
$$

**分解**：
$$
BS = Reliability - Resolution + Uncertainty
$$

- Reliability：可靠性（越低越好）
- Resolution：分辨率（越高越好）
- Uncertainty：数据固有不确定性

#### 2.1.3 NLL（负对数似然）

$$
NLL = -\frac{1}{N} \sum_{i=1}^{N} \log(p_{i, y_i})
$$

**特点**：
- 训练常用损失函数
- 对校准和区分性都敏感

### 2.2 ECE变体

| 变体 | 公式特点 | 适用场景 |
|------|----------|----------|
| **Classwise-ECE** | 对每个类别单独计算 | 类别不平衡 |
| **Adaptive-ECE** | 自适应分桶边界 | 避免空桶 |
| **Thresholded-ECE** | 只考虑高置信度区域 | 关注高置信度校准 |
| **Margin-ECE** | 基于置信度差距分桶 | 关注决策边界 |

### 2.3 度量选择指南

```
选择校准度量
    ↓
是否需要分解分析？
    ├── 是 → Brier Score（可分解）
    └── 否 → 继续
         ↓
是否类别不平衡？
    ├── 是 → Classwise-ECE
    └── 否 → 标准ECE
```

## 三、校准方法分类

### 3.1 后处理校准（Post-hoc）

在训练完成后进行，无需重新训练模型。

#### 3.1.1 参数化方法

| 方法 | 参数数量 | 特点 | 适用 |
|------|----------|------|------|
| **Temperature Scaling** | 1 | 简单高效 | 通用首选 |
| **Vector Scaling** | C | 每类独立缩放 | 多分类校准 |
| **Matrix Scaling** | C×C | 类间相关建模 | 复杂校准需求 |
| **MS-ODIR** | C×C + C | 结合检测重校准 | 目标检测 |

#### 3.1.2 非参数化方法

| 方法 | 特点 | 优缺点 |
|------|------|--------|
| **Histogram Binning** | 直方图映射 | 简单但离散化 |
| **Isotonic Regression** | 单调回归 | 灵活但易过拟合 |
| **Bayesian Binning** | 贝叶斯分桶 | 理论保证但复杂 |
| **Platt Scaling** | Sigmoid拟合 | 适合二分类 |

#### 3.1.3 后处理方法对比

```
准确性要求 vs 计算资源
    ↓
资源充足？
    ├── 否 → Temperature Scaling（最简单）
    └── 是 → 
         ↓
数据充足？
    ├── 是 → Isotonic Regression
    └── 否 → Temperature Scaling（更稳定）
```

### 3.2 训练时校准（In-training）

在模型训练过程中引入校准约束。

#### 3.2.1 损失函数修改

| 方法 | 原理 | 效果 |
|------|------|------|
| **Label Smoothing** | 软化硬标签 | 改善过度自信 |
| **Focal Loss** | 降低易样本权重 | 减少高置信度 |
| **MMCE** | 最大均值校准误差 | 直接优化ECE |
| **FBCE** | 基于流的校准 | 端到端校准 |

#### 3.2.2 正则化技术

| 技术 | 原理 | 适用 |
|------|------|------|
| **更大Weight Decay** | 强正则化 | 通用 |
| **Dropout** | 随机失活 | 不确定性估计 |
| **Mixup** | 样本混合 | 间接改善 |
| **CutMix** | 区域混合 | 视觉任务 |

#### 3.2.3 网络架构设计

| 设计 | 原理 | 代表工作 |
|------|------|----------|
| **Stochastic Depth** | 随机深度 | 减少过度自信 |
| **MC Dropout** | 推理时dropout | 不确定性量化 |
| **Deep Ensemble** | 多模型集成 | 校准与区分性兼顾 |

## 四、不同任务的校准

### 4.1 图像分类

**挑战**：
- 标准ECE适用
- 类别不平衡问题
- 细粒度分类的特殊性

**方法**：
- Temperature Scaling为首选
- Classwise-ECE评估

### 4.2 目标检测

**挑战**：
- 分类+定位联合置信度
- 背景类处理
- IoU与置信度的关系

**ECE扩展**：

$$
Detection\text{-}ECE = \sum_{m} \frac{|B_m|}{N} \cdot |conf(B_m) - AP(B_m)|
$$

其中AP为平均精度。

**方法**：
- MS-ODIR（检测专用后处理）
- 分类和定位分别校准

### 4.3 语义分割

**挑战**：
- 像素级预测
- 类别不平衡更严重
- 空间相关性

**方法**：
- Pixel-wise Temperature Scaling
- 空间感知的校准

### 4.4 其他任务

| 任务 | 校准挑战 | 解决方案 |
|------|----------|----------|
| **实例分割** | 实例级置信度 | 检测ECE扩展 |
| **姿态估计** | 关键点置信度 | 热力图校准 |
| **深度估计** | 回归任务 | 分箱化或概率回归 |

## 五、前沿研究方向

### 5.1 大语言模型校准

```
LLM校准新挑战
├── 生成任务无固定类别
│   └── Flex-ECE等变体
├── 长文本生成评估难

├── RLHF后校准退化
```

**挑战**：
- 数据分布异构

**方向**：
- 联邦校准算法
- 差分隐私与校准的权衡

**发现**：
- 自监督预训练模型校准性能不一

- 源域校准 → 目标域校准
- 域漂移对校准的影响

## 六、关键见解与总结

### 6.1 核心结论

1. **深度学习模型普遍存在校准问题**：准确率提升常伴随校准退化
2. **Temperature Scaling是实用首选**：简单、有效、不改变预测
3. **任务特性决定校准策略**：分类、检测、分割方法各异
4. **训练时与后处理可结合**：Label Smoothing + Temperature Scaling是黄金组合

### 6.2 实践建议

| 阶段 | 建议 |
|------|------|
| 训练前 | 考虑Label Smoothing或Focal Loss |
| 训练中 | 监控验证集ECE |
| 训练后 | 应用Temperature Scaling |
| 部署时 | 根据场景决定是否校准 |

### 6.3 开放问题

- 如何设计理论上更优的校准度量？
- 如何在保持准确率的同时优化校准？
- 大模型的校准有何特殊性？
- 校准与公平性、鲁棒性的关系？
