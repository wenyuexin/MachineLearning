# HippoRAG

HippoRAG（NeurIPS 2024）是一个受神经生物学启发的长时记忆框架，旨在让 LLM 像人脑一样持续整合新知识并实现多跳推理。其核心创新在于将知识图谱构建与图检索算法相结合，模拟人脑海马体（Hippocampus）的记忆索引机制。

---

## 一、动机：为什么需要 HippoRAG？

传统 RAG 的问题：

- **单次检索**：每次查询独立检索文本块，无法跨文档整合知识
- **无持续学习**：新知识需要重新索引整个语料库，无法增量整合
- **无法多跳推理**：只做语义匹配，不能关联分散在不同文档中的实体关系

人脑的海马体记忆索引理论提供了一种解决方案：大脑不存储所有细节，而是存储一个**索引**，指引如何访问完整的记忆。HippoRAG 将这一机制引入 RAG。

---

## 二、核心架构

HippoRAG 的架构直接映射人脑长时记忆的三个功能区域：

```
┌──────────────────────────────────────────────────────────┐
│                   HippoRAG 架构                            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   ┌──────────────────┐    ┌──────────────────┐          │
│   │   LLM (新皮层)    │    │   检索编码器      │          │
│   │   实体/关系抽取   │    │   查询编码 +      │          │
│   │   最终答案生成    │    │   种子节点选择    │          │
│   └────────┬─────────┘    └────────┬─────────┘          │
│            │                       │                     │
│            ▼                       ▼                     │
│   ┌──────────────────────────────────────────┐          │
│   │         知识图谱 (海马体)                   │          │
│   │   实体节点 + 关系边 + 文本块引用            │          │
│   │   + Personalized PageRank 检索            │          │
│   └──────────────────────────────────────────┘          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 组件映射

| 人脑区域 | HippoRAG 组件 | 功能 |
|---------|--------------|------|
| **新皮层 (Neocortex)** | LLM | 理解输入、抽取实体和关系、生成最终答案 |
| **海马体 (Hippocampus)** | 知识图谱 + PPR 算法 | 存储记忆索引，通过关联性检索相关信息 |
| **海马旁回 (Parahippocampal)** | 检索编码器 | 连接 LLM 和知识图谱，将查询映射到图检索 |

## 三、工作流程

### 阶段一：离线索引（构建知识图谱）

```
输入语料 → LLM 抽取实体+关系 → 构建开放知识图谱 → 存储节点+边+文本引用
```

1. 对每个文档/文本块，LLM 抽取其中的**命名实体**（人物、地点、概念等）和**关系**（实体间的语义连接）
2. 将实体作为图中的节点，关系作为边，构建一个**无模式（schemaless）知识图谱**
3. 每个节点保留其来源文本块的引用，支持检索时回溯到原始文档

不同于传统知识图谱需要预定义 schema，HippoRAG 的图谱是开放式的——实体和关系由 LLM 在索引时动态确定，不需要预先设计本体。

### 阶段二：在线检索（查询与推理）

```
用户查询 → 编码查询向量 → PPR 图检索 → 返回相关文本块 → LLM 生成答案
```

1. **查询编码**：用检索编码器将用户问题编码为向量
2. **种子节点选择**：向量检索找到与查询最相关的 k 个实体节点（种子节点）
3. **Personalized PageRank (PPR)**：
   - 以种子节点为起点，在知识图谱上运行 PPR 算法
   - PPR 利用图的拓扑结构，找到与种子节点**关联性强**的其他节点（即使它们与查询的向量相似度不高）
   - 这正是 HippoRAG 实现多跳推理的关键机制
4. **文本召回**：根据 PPR 排序结果，回溯到关联的原始文本块
5. **答案生成**：LLM 基于召回文本生成回答

## 四、HippoRAG 的优势

### 1. 跨文档知识整合

传统 RAG 的检索范围限于单次向量匹配。HippoRAG 通过知识图谱将分散在不同文档中的实体关联起来。例如：

```
文档 A："阿司匹林用于治疗头痛"
文档 B："头痛是 C 故障的常见表现"
文档 C："C 故障场景下应避免采用 D 方案"

查询："A 系统在 C 故障场景下是否安全？"

HippoRAG 通过图路径：A→B→C→D，关联了三篇文档
```

### 2. 多跳推理

PPR 算法天然支持多跳——从种子节点出发，沿着图的边逐步扩散，可以找到间接关联的信息。相比传统 RAG 需要多次独立检索，HippoRAG 一次 PPR 即可完成多跳推理。

### 3. 持续学习

新文档到来时，只需增量抽取实体和关系加入已有知识图谱，不需要重新索引整个语料库。这与海马体的持续记忆整合机制一致，避免了灾难性遗忘。

### 4. 关联性检索（Associativity）

HippoRAG 是唯一通过 PPR 在显式构建的知识图谱上实现**关联性检索**的 RAG 框架，即能够像人脑一样从一条信息联想到另一条看似不直接相关但通过中间节点连接的信息。

## 五、HippoRAG 2（2025）

HippoRAG 2 在原版基础上做了三项关键改进：

### 1. 概念与上下文融合

原版图谱只存储实体和关系三元组，丢失了文本块的上下文信息。HippoRAG 2 同时将文本块的嵌入向量（上下文）也关联到图谱节点上，保留概念的结构化关系和语义上下文。

```
原版：node(阿司匹林) --[治疗]--> node(头痛)
HippoRAG 2：node(阿司匹林) --[治疗]--> node(头痛)
             node(阿司匹林) ── 关联文本块嵌入 ──→ passage_embedding_1
```

### 2. 上下文感知检索

利用图谱结构进行超越孤立节点的检索，在图搜索过程中同时利用结构信息和语义信息。

### 3. 识别记忆（Recognition Memory）

引入类似人脑"熟悉感"的识别机制来改进种子节点选择——不仅看向量相似度，还评估该节点在当前上下文中的"熟悉程度"，提高种子节点选择的准确率。

## 六、GraphRAG vs HippoRAG

| 维度 | GraphRAG | HippoRAG |
|------|---------|----------|
| **核心动机** | 通过知识图谱增强单次检索的相关性 | 模拟人脑长时记忆，实现持续学习和关联检索 |
| **图构建方式** | 通常基于预定义 schema 或领域本体 | 无模式（schemaless），LLM 动态抽取 |
| **检索算法** | 子图查询（SPARQL/Cypher） | Personalized PageRank |
| **持续学习** | 需要重索引或手动更新图谱 | 增量式，新知识直接加入图谱 |
| **多跳推理** | 通过图查询路径 | 通过 PPR 在图上的传播 |
| **记忆机制** | 无，聚焦单次检索质量 | 模拟海马体记忆索引理论 |
| **代表文献** | Microsoft GraphRAG (2024) | HippoRAG, NeurIPS 2024 |

## 七、常见面试题

### 面试题 1：HippoRAG 是如何实现跨文档知识整合的？

**参考答案**：  
HippoRAG 在索引阶段用 LLM 从所有文档中抽取实体和关系，构建一个统一的知识图谱。当查询到来时，先通过向量检索找到种子实体节点，然后用 Personalized PageRank 在图谱中传播——不仅找到直接相关的节点，还能沿关系边扩散到间接相关的其他节点。由于图谱是跨文档构建的，PPR 可以自然地将不同文档中的相关信息关联起来。

### 面试题 2：Personalized PageRank 在 HippoRAG 中的作用是什么？

**参考答案**：  
PPR 是 HippoRAG 实现多跳推理和关联性检索的核心算法。它以种子节点为起点（赋予更高的初始权重），在图谱上迭代传播权重。与普通 PageRank 不同，PPR 偏向于与种子节点相关的区域。这使得系统能发现与查询词汇上不相似但在语义关系上相关的信息——例如查询"阿司匹林安全性"，PPR 可以通过"阿司匹林→头痛→高血压→禁忌药"的路径找到相关的安全警告信息。

### 面试题 3：HippoRAG 和传统 RAG 在多跳推理能力上的本质区别是什么？

**参考答案**：  
传统 RAG 的多跳需要多次独立检索（先是 first-hop 检索，用结果做 second-hop 检索），每次检索互不关联，且错误会逐跳累积。HippoRAG 的 PPR 一次检索即可完成多跳：种子节点启动后，PPR 算法自动沿着关系边在图谱中传播，一跳、两跳、三跳的关联信息在一次计算中被同时考虑。这种"联想"机制更接近人脑的思维方式。
