## 序列建模：Mamba vs Transformer

- Mamba（状态空间模型 SSM）与 Transformer 的核心复杂度差异：O(L) vs O(L²)
- Mamba 的隐藏状态记忆性，如何避免 RNN 的梯度消失问题
- 双向建模：正向 + 反向状态空间模型捕捉前后文依赖
- 适用场景：长序列（数百帧）推理，显存受限环境下优势明显

### 面试题

> Q: 为什么 Mamba 在处理长序列时比 Transformer 更有优势？

**参考答案**：  
Mamba 基于状态空间模型（SSM），计算复杂度为 O(L)，而 Transformer 的自注意力为 O(L²)。在处理数百帧的长序列时，Mamba 显存占用更低、推理速度更快。此外，Mamba 的隐藏状态具有记忆性，可以高效捕捉逐帧之间的时间依赖关系，且不像 RNN 那样存在梯度消失问题。

> Q: 双向状态空间模型与单向相比有什么优势？

**参考答案**：  
双向建模沿序列正反两个方向分别计算隐藏状态，使当前位置同时融合过去和未来的信息。对于周期性的序列数据（如时间序列预测、传感器数据），双向模型可以更准确地插值或补全缺失的片段。

## 模型轻量化与推理优化

- **模型蒸馏**：大模型（如 13B）→ 小模型（如 7B），保留核心能力的同时降低推理成本
- **量化**：INT8 / FP16 量化，减少模型体积和推理延迟
- **结构化剪枝**：移除冗余层或通道，裁剪不用于特定任务的模块
- **降维**：减少隐藏状态维度（如 512 → 256），降低计算量
- **推理框架**：ONNX Runtime / TensorRT 导出，算子融合，半精度计算
- **流式推理**：连续帧场景下只更新状态而不重复计算整个序列历史
- **流水线并行**：CPU 读取下一帧同时 GPU 推理当前帧，隐藏 I/O 延迟
- **GPU 加速预处理**：图像归一化、resize 等操作从 CPU 迁移到 GPU，使用 pinned memory 加速数据拷贝

### 面试题

> Q: 模型轻量化有哪些常用手段，各自的优缺点是什么？

**参考答案**：  
常见手段包括：（1）模型蒸馏——用大模型输出训练小模型，效果好但需要预训练好的教师模型；（2）量化（INT8/FP16）——直接压缩权重精度，几乎不损失精度但需要硬件支持；（3）结构化剪枝——移除冗余通道或层，可大幅减少参数量但需要重新微调恢复精度；（4）降维——减少隐藏层维度，简单直接但可能损失表达能力。实践中通常组合使用。

> Q: 流式推理与普通批处理推理的区别是什么？

**参考答案**：  
流式推理适用于连续帧或流式输入场景。每次只处理当前帧（或 token），同时维护一个内部状态（如 SSM 的隐藏状态或 Transformer 的 KV Cache），避免对历史序列的重复计算。相比批处理，流式推理延迟更低、显存占用更稳定，但无法利用 batch 内的并行加速。

## 半监督学习

- **伪标签生成**：用有监督训练的最优模型为无标签数据生成预测作为标签
- **置信度筛选**：只保留高置信度样本（如预测概率 > 0.9 的区域占比 > 70%）
- **渐进式训练**：先加入高质量伪标签训练，再用新模型生成剩余样本的伪标签（自训练迭代）
- **一致性正则化**：对同一输入的不同增强版本要求模型预测一致
- **伪标签损失加权**：降低伪标签在联合训练中的损失权重（如设为 0.5），抑制噪声累积
- **多教师集成**：多个不同初始化的教师模型投票，只有一致的区域才采用

### 确证偏差（Confirmation Bias）

模型用自己生成的错误伪标签不断强化错误预测，导致错误被放大且难以纠正。

**缓解策略**：
- 噪声注入：对伪标签添加形态学腐蚀/膨胀等随机扰动
- 不一致丢弃：同一无标签样本的不同增强版本预测不一致时弃用该伪标签
- 课程学习：初期只使用最高置信度的伪标签，随模型能力提升逐步放宽阈值

### 面试题

> Q: 半监督学习中如何保证伪标签的质量？

**参考答案**：  
常用方法包括：（1）置信度筛选，只保留高置信度样本（如预测概率 > 0.9）；（2）渐进式训练，分批加入伪标签而非一次性全部使用；（3）一致性正则化，不同增强版本的预测应一致；（4）降低伪标签的损失权重，避免噪声主导训练；（5）多教师集成投票；（6）人工抽检，随机校验发现系统性偏差后调整阈值。

> Q: 什么是自我训练中的确证偏差？如何缓解？

**参考答案**：  
确证偏差是指模型用自己的错误伪标签不断强化同样的错误。缓解策略包括：对伪标签添加扰动增加鲁棒性、丢弃预测不一致的样本、课程学习逐步放宽置信度阈值、多教师集成减少单一模型的偏差。

## 模型评估

- 混淆矩阵及衍生指标（Precision、Recall、F1、AUC-ROC）
- AUC 的物理意义，为什么对类别不平衡不敏感？
- 偏差与方差的权衡（Bias-Variance Tradeoff）
- 交叉验证的不同方式
- **Dice 系数**：分割任务中衡量区域重叠度，2×|A∩B|/(|A|+|B|)
- **HD95（Hausdorff Distance 95th percentile）**：衡量两个点集边界之间最大距离的 95% 分位数，比最大 Hausdorff 更鲁棒（忽略离群点），值越小表示边界越吻合
- **PSNR / SSIM**：图像重建/超分任务中的常用指标

### 面试题

> Q: Dice 和 HD95 这两个指标关注的分割质量维度有何不同？

**参考答案**：  
Dice 关注的是整体区域重叠程度，对大面积重合敏感但可能忽略边界细节。HD95 关注的是边界最大误差（95 分位数），对边缘定位精度更敏感。两者互补：Dice 高 + HD95 低 = 整体和边界都好；Dice 高但 HD95 高 = 区域大致正确但边界有较大偏差。
