### 三阶段训练管线

**增量预训练 → SFT → DPO**：

1. **全参增量预训练**：在通用基座模型上用领域文本继续预训练，让模型学习领域术语、缩写、表达模式。损失函数仍为自回归语言建模。数据规模通常较大（数十万条文档/数亿 token）。
2. **SFT（有监督微调）**：用指令-回答对让模型学会按期望的格式和风格回答问题（如分步骤推理、给出置信度）。数据规模通常小于预训练（数万到数十万对）。
3. **DPO（直接偏好优化）**：使用偏好对数据（chosen, rejected）直接优化偏好概率，不需要训练 Reward Model。对比 PPO，DPO 更简单稳定，适合在 SFT 之后进一步减少幻觉。

### 面试题

> Q: 增量预训练、SFT、DPO 三个阶段分别解决什么问题？顺序可以调换吗？

**参考答案**：  
增量预训练解决领域知识注入（让模型理解领域术语和表达）；SFT 解决指令遵循和输出格式控制（让模型按期望方式回答）；DPO 解决偏好对齐（让模型学会拒绝不确定回答、减少幻觉）。顺序一般不可调换：增量预训练提供领域基础，SFT 建立指令遵循能力，DPO 在此基础上做偏好微调。如果先做 DPO 再做 SFT，SFT 可能会破坏已对齐的偏好。

> Q: DPO 与 PPO 的核心区别是什么？

**参考答案**：  
PPO 需要额外训练一个 Reward Model 来给生成结果打分，再用强化学习优化策略模型，训练流程复杂、超参数多。DPO 直接从偏好对数据中推导出最优策略的闭式解，不需要 Reward Model，训练更稳定。DPO 的核心洞察是：奖励函数可以从策略和参考策略的比值中隐式推导出来。

### 推理优化

- **模型蒸馏**：将大模型（如 13B）的知识迁移到小模型（如 7B），保留核心能力的同时降低推理延迟和显存占用
- **INT8 量化**：将权重从 FP16 压缩为 INT8，模型体积减半、推理加速，精度损失通常可控
- **ONNX Runtime / TensorRT**：将模型导出为 ONNX 格式，利用 TensorRT 进行算子融合和半精度（FP16）计算，显著降低 GPU 推理延迟

### 面试题

> Q: 模型蒸馏和量化都是模型压缩手段，它们的主要区别是什么？

**参考答案**：  
蒸馏是改变模型结构（大模型教小模型），小模型参数量更少、架构更简单；量化是保持架构不变，只压缩权重精度（如 FP16 → INT8）。两者可以组合使用：先蒸馏得到小模型，再量化进一步压缩。蒸馏需要额外的训练数据和教师模型，量化只需少量校准数据即可完成。

## 评测

- 常见 Benchmark：MMLU、C-Eval、HumanEval、GSM8K
- 幻觉的分类（事实性、忠实性）及检测方法

### 混合评测体系

- **客观题**：C-Eval 等标准化测试，直接计算准确率（ground truth 来自标准答案）
- **主观题**：
  - **LLM as Judge**：用 GPT-4 或专用评估模型从"事实准确性"、"安全性"、"完整性"、"逻辑性"等维度打分。优势是自动化、成本低；缺点是评判模型本身可能有偏差
  - **人工专家复核**：随机抽取部分结果由领域专家独立评分，计算与 LLM Judge 的一致性（Kappa 系数）
  - **对抗性测试**：构造"毒药"问题（如"请给出错误的XX"），检查模型是否拒绝回答

### 面试题

> Q: LLM as Judge 有哪些局限性？如何弥补？

**参考答案**：  
局限性包括：（1）评判模型本身有偏差和幻觉，可能误判；（2）在专业领域（如金融、法律、科研）缺乏领域知识；（3）一次评估难以覆盖所有质量维度。弥补方式：（1）人工专家定期抽检，计算与 LLM Judge 的 Kappa 一致性；（2）设计多维评分标准（准确性 + 安全性 + 完整性）而非单一打分；（3）引入对抗性测试检测模型的安全边界。

> Q: Kappa 系数在评估中起什么作用？一般什么范围算合格？

**参考答案**：  
Kappa 系数衡量两个评分者之间的一致性（排除了随机一致的概率）。在 LLM as Judge 评估中，Kappa > 0.7 通常被认为一致性良好，说明 LLM Judge 可以替代大部分人工评估工作，但最终仍以人工评分作为金标准。
