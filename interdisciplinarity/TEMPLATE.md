# 跨学科笔记模板

> 本模板用于规范 `interdisciplinarity/` 下各主题笔记的结构，确保跨学科映射的一致性与可操作性。

---

## 标题格式

```
# [外部学科概念]：[核心张力或问题简述]
```

示例：
- `# 组织双元性：流程效率与适应柔性的平衡`
- `# 预测编码：大脑的自监督假设与 LLM 预训练的同构性`

---

## 正文结构

### 1. 学科背景（What）

简述该概念在外部学科中的起源、核心定义与理论脉络。避免长篇大论，聚焦对 AI/ML 映射有直接启发性的部分。

### 2. 核心张力（Tension）

**回答映射方法论第 1 问**：这个学科的核心张力是什么？

- 用 1-2 句话提炼对立面（如：效率 vs 柔性、还原论 vs 整体论）
- 说明该张力在该学科中的历史演进或代表性理论

### 3. AI/ML 中的同构表现（Isomorphism）

**回答映射方法论第 2 问**：该张力在 LLM / ML 系统中有无同构表现？

- 指出 AI 系统中对偶的张力或现象
- 提供具体例子（模型、论文、工程实践均可）

### 4. 外部学科的解决方案（Solutions）

**回答映射方法论第 3 问**：该学科发展出了哪些解决方案或理论模型？

- 列举 2-4 个关键理论、模型或机制
- 说明其适用条件与局限性

### 5. 工程转译（Translation）

**回答映射方法论第 4 问**：这些解决方案能否转译为工程架构或训练策略？

- 提出具体的转译方向（算法、架构、流程、评估指标）
- 区分"已验证的映射"与"待探索的假设"

## 写作原则

1. **拒绝修辞性类比**：不说"这就像…"，而是说明"同构性体现在…"或"差异在于…"
2. **标注确定性等级**：用"已验证""高置信度假设""推测性"区分论断强度
3. **保持双向开放**：不只谈"外部学科如何启发 AI"，也谈"AI 进展如何反过来检验或修正外部学科理论"
4. **链接优先于复述**：对已有成熟论述的概念，优先链接到纵向技术目录的详细笔记，而非在本笔记中重复
