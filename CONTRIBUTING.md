# 仓库组织与贡献指南

本文件定义仓库的组织规则与新增内容的操作规范。

---

## 一、层级设计

仓库按纵向技术栈分层，辅以横向跨学科层：

| 层级 | 定位 | 目录 |
|---|---|---|
| 基础方法层 | 经典算法与理论基础 | traditional-ml、deep-learning、reinforce-learning |
| 核心领域层 | 仓库主线 | llm、cv、knowledge-graph |
| 应用与集成层 | 技术落地与组合 | rag、agentic、world-models、embodied-intelligence |
| 基础设施层 | 训练与工程支撑 | training-infra |
| 跨学科层 | 横向交叉视角 | interdisciplinarity |
| 支撑资源 | 辅助资料 | learning-materials、assets |

## 二、目录与文件命名

- 一级目录：英文短名，语义自解释
- 二级及以下目录：`编号-英文短名` 格式（如 `01-foundations/`），编号保证排序稳定
- 目录名不使用中文
- 笔记文件：英文短名，语义自解释（如 `lora.md`、`rlhf-overview.md`）
- 论文笔记：`论文简称_arXiv编号.md`（如 `ReAct_2210.03629.md`）或 `论文简称_会议年份.md`（如 `BeingH05_ICLR2026.md`）

## 三、展开深度

根 README 的目录树展开深度按内容活跃度区分：

- **核心领域**（traditional-ml、deep-learning、reinforce-learning、llm、cv、rag、agentic）展开到 4 级——内容充实、更新频繁，需要更细粒度的导航
- **次要领域**（knowledge-graph、embodied-intelligence、world-models、training-infra、interdisciplinarity、learning-materials）展开到 3 级——内容量较少或更新节奏较慢
- 各一级目录有自己的 README，承载完整目录结构（不受根 README 深度限制）

## 四、README 规范

每个一级目录下必须有 README.md。二级及更深层目录如有较多内容，也应创建 README。

### 组织规则

- **标题**：中英文并列（如 `# LLM (大语言模型)`）
- **目录结构**：必填，展开到最深层的子目录，不受根 README 深度限制
- **更新时间**：末尾标注 `*最后更新: YYYY-MM-DD*`，每次修改时同步更新
- **跨目录引用**：基础内容放在所属领域目录下，其他目录通过相对路径引用，避免重复维护
- **跨学科目录特殊规则**：排序逻辑和定位放在目录结构之前（先建立框架再列细节）；各领域概览中用映射表关联技术目录；子目录按 **个体→系统→群体→制度** 递进排列：
  ```
  个体层面   01 认知科学与神经科学
            02 语言学与语用学
            03 心灵哲学
  系统层面   04 控制论与系统论
            05 复杂系统与涌现
  群体层面   06 社会学与组织管理
            07 经济学与博弈论
  制度层面   08 法学与治理
  ```

### 推荐内容要素

不同类型 README 建议包含的章节，可根据实际需要增减：

**一级目录（技术类）**：目录结构 + 开源仓库与工具存放指南 + 学习路径 + 相关资源

**一级目录（跨学科类）**：一句话定位引言 + 排序逻辑 + 定位 + 目录结构 + 各领域概览 + 与纵向技术栈的关系

**子目录（论文集）**：研究主题 + 创建日期 + 论文列表 + 主题分类 + 阅读建议 + 工具与资源 + 相关资源 + 参考文献 BibTeX。论文列表推荐用表格（论文、文档链接、arXiv ID、来源、核心贡献、重要性），重要性可用三级标注（⭐⭐⭐ 必读 / ⭐⭐ 值得关注 / ⭐ 简单参考）

**子目录（技术主题）**：目录结构 + 领域特色内容 + 相关资源。目录结构推荐用表格（目录、说明），可加"状态"列标注填充进度

## 五、笔记写作建议

以下为推荐做法，不强制：

- 知识性笔记：可包含数学公式（LaTeX）、代码块、表格等
- 论文笔记：建议包含核心贡献、方法描述、关键结果，不必逐段翻译
- README 中如有流程或阶段关系，可用代码块/ASCII 图展示

## 六、新增内容操作指引

### 新增笔记

1. 确定所属领域，找到对应一级目录
2. 根据主题找到最匹配的编号子目录（如 LLM 训练相关 → `llm/03-training/`）
3. 按第二章规则命名文件
4. 如无法归入现有子目录，可新建子目录（遵循 `编号-英文短名` 格式），并同步更新该领域的 README

### 新增子目录

1. 确定在层级中的位置和编号（接续现有编号）
2. 创建目录并添加 `.gitkeep` 占位
3. 更新该一级目录的 README 目录结构
4. 如展开深度达到根 README 的展示范围，同步更新根 README

### 新增一级目录

1. 评估该目录属于哪个层级（基础方法 / 核心领域 / 应用与集成 / 基础设施 / 跨学科 / 支撑资源）
2. 创建目录、README.md、`.gitkeep`
3. 按层级顺序更新本文件的层级设计表
4. 更新根 README 的层级关系、知识体系（Mermaid 图）、目录结构三个章节

## 七、其他规则

- **空目录**：使用 `.gitkeep` 占位，确保 Git 跟踪
