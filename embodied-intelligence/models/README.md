# 模型架构

本目录整理具身AI/VLA模型的架构设计和代表模型。

## 模型分类

### 1. 大语言模型（LLM）

用于高层任务规划、代码生成、知识推理。

| 模型 | 特点 | 应用场景 |
|------|------|----------|
| GPT系列 | 强大的语言理解和生成能力 | 任务规划、代码生成 |
| LLaMA | 开源，可微调 | 定制化应用 |
| Deepseek-R1 | 推理能力强 | 复杂任务分解 |

### 2. 多模态大语言模型（MLLM/VLM）

桥接视觉和语言，具备跨模态理解能力。

| 模型 | 特点 | 链接 |
|------|------|------|
| GPT-4o | 多模态融合 | OpenAI |
| Gemini-1.5 | 长上下文，多模态 | Google |
| Qwen-VL | 开源中文多模态 | [github.com/QwenLM/Qwen-VL](https://github.com/QwenLM/Qwen-VL) |
| CLIP | 视觉-语言对齐 | [github.com/openai/CLIP](https://github.com/openai/CLIP) |
| SigLIP | 改进的视觉编码器 | Google |

### 3. 视觉-语言-动作模型（VLA）

端到端从视觉语言输入输出机器人动作。

#### 发展阶段

| 阶段 | 时间 | 核心问题 | 代表模型 |
|------|------|----------|----------|
| 萌芽期 | 2023.7前 | 如何让机器人听懂人话 | RT-1, CLIPort, GATO, VIMA |
| 探索期 | 2023.7-2024 | 如何让机器人具备常识 | RT-2, Octo, OpenVLA |
| 发展期 | 2024至今 | 如何既聪明又快 | π0, GR00T, Hume |

#### 代表模型详解

| 模型 | 架构特点 | 动作输出 | 开源 |
|------|----------|----------|------|
| RT-1 | Transformer, 离散动作 | 离散Token | 部分 |
| RT-2 | 继承VLM权重 | 离散Token | 否 |
| OpenVLA | 基于Llama | 连续动作 | [是](https://github.com/openvla/openvla) |
| Octo | 扩散策略 | 连续动作 | [是](https://github.com/octo-models/octo) |
| π0 | 流匹配 | 连续动作 | 否 |
| GR00T | 视频预训练 | 连续动作 | 部分 |

### 4. 世界模型

预测未来状态，支持规划和学习。

| 类别 | 代表模型 | 特点 |
|------|----------|------|
| 潜空间模型 | DreamerV3 | 成熟稳定，计算高效 |
| JEPA架构 | I-JEPA | 避免重建损失 |
| Transformer WM | Genie, TWM | 长程依赖建模强 |
| 扩散世界模型 | Sora, Cosmos | 生成质量最高 |

## VLA模型架构

### 核心组件

```
┌─────────────────────────────────────────────┐
│                 VLA模型架构                   │
├─────────────────────────────────────────────┤
│  观测编码器 (Observation Encoder)            │
│  ├── 视觉编码 (ViT/CNN)                      │
│  ├── 语言编码 (T5/BERT)                      │
│  └── 多模态传感器 (深度/触觉/力觉)            │
├─────────────────────────────────────────────┤
│  特征推理主干 (Backbone)                     │
│  ├── Standard Transformer                   │
│  ├── DiT (Diffusion Transformer)            │
│  ├── Mamba                                  │
│  └── MoE (Mixture of Experts)               │
├─────────────────────────────────────────────┤
│  动作解码器 (Action Decoder)                 │
│  ├── 离散Token (Binning)                    │
│  ├── Diffusion Policy                       │
│  ├── Flow Matching                          │
│  └── Action Chunking                        │
└─────────────────────────────────────────────┘
```

### 分层架构

```
┌──────────────────────────────────────┐
│        System 2 (大脑)                │
│  预训练VLM/VLA, 慢思考, 规划          │
│  输出: 子任务/中间表示/目标           │
└──────────────┬───────────────────────┘
               │ 通信原语
               │ (文本/动作轨迹/隐特征)
┌──────────────▼───────────────────────┐
│        System 1 (小脑)                │
│  小型策略网络, 快执行                 │
│  输出: 高频控制信号 (50Hz+)           │
└──────────────────────────────────────┘
```
