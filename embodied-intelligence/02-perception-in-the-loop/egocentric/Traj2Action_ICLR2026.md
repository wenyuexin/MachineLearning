# Traj2Action：通过轨迹对齐从人手到机器臂的技能迁移

**论文信息**
- 论文标题：Traj2Action: A Co-Denoising Framework for Trajectory-Guided Human-to-Robot Skill Transfer
- 中文标题：Traj2Action：轨迹引导的人机技能迁移共去噪框架
- 作者：Han Zhou, Jinjin Cao, Liyuan Ma, Xueji Fang, Guo-Jun Qi
- 会议：ICLR 2026
- arXiv: [2510.00491](https://arxiv.org/abs/2510.00491)
- OpenReview: [vPKy3qQG3W](https://openreview.net/forum?id=vPKy3qQG3W)

> **核心创新**：使用3D轨迹作为统一中间表示，通过共去噪框架弥合人-机器人形态差距，实现高达27%的性能提升

---

## 一、论文整体思路

### 1.1 研究背景

学习多样化的机器人操作技能面临严重的数据瓶颈：

**核心问题**：
- 遥操作演示成本高、难以扩展
- 人类视频作为可扩展的替代方案
- 但人-机器人之间存在显著的**形态差距(Morphological Gap)**

**形态差距具体表现**：
- 人手与机器臂结构差异大
- 自由度、运动范围不同
- 难以直接映射动作

### 1.2 核心创新

Traj2Action提出使用**操作端点的3D轨迹**作为统一中间表示，解决形态差距问题。

**核心思路**：
```
人类视频 → 3D轨迹(统一表示) → 机器人动作
           形态无关         形态特定
```

**两大关键问题**：
1. 如何统一人类和机器人演示（结构差异显著）？
2. 如何将统一知识有效集成到机器人学习过程？

## 二、方法架构

### 2.1 整体框架

```
┌──────────────────────────────────────────────────────────┐
│                   Traj2Action 框架                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  输入: 多视角图像 + 语言指令                              │
│  ┌────────────────────────────────────────────┐         │
│  │  Left View + Ego View + Language           │         │
│  └────────────────────────────────────────────┘         │
│                        │                                 │
│                        ▼                                 │
│  阶段1: 轨迹专家 (Trajectory Expert)                     │
│  ┌────────────────────────────────────────────┐         │
│  │ • 训练于人类+机器人数据                     │         │
│  │ • 预测粗糙的3D轨迹                         │         │
│  │ • 高层运动规划                             │         │
│  └────────────────────────────────────────────┘         │
│                        │                                 │
│                        ▼                                 │
│  阶段2: 共去噪框架                                        │
│  ┌────────────────────────────────────────────┐         │
│  │  轨迹条件 → 细粒度机器人动作生成            │         │
│  │  • 方向 (orientation)                       │         │
│  │  • 夹持器状态 (gripper state)               │         │
│  └────────────────────────────────────────────┘         │
│                        │                                 │
│                        ▼                                 │
│  输出: 机器人动作序列 (Δd, Δθ, ΔGrip)                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 2.2 统一轨迹空间

**核心思想**：3D轨迹作为形态无关的统一表示

**轨迹表示**：
- 操作端点的3D位置序列
- 描述"在哪里做什么"
- 不依赖具体形态结构

**统一策略**：
- 将人类和机器人运动表示到统一的轨迹空间
- 轨迹专家在两种数据上联合训练
- 学习形态无关的运动知识

### 2.3 共去噪框架 (Co-Denoising)

**技术原理**：

**损失函数**：
```
ℒ_action(θ) = E[||π_θ(a_τ, T_τ, τ, I_t, P, q_t) - (z - a*)||²]
```

其中：
- `τ`: 轨迹条件
- `a_τ`: 轨迹引导的动作
- `T_τ`: 轨迹时间信息
- `I_t`: 图像观测
- `P`: 语言指令
- `q_t`: 机器人状态

**去噪过程**：
1. 轨迹作为条件指导
2. 同时去噪轨迹和动作
3. 生成细粒度机器人动作

**生成内容**：
- 位置增量 (Δd)
- 方向增量 (Δθ)
- 夹持器状态 (ΔGrip)

### 2.4 数据收集流程

**跨形态演示收集**：
- 人类操作视频
- 机器人遥操作数据
- 对应的轨迹标注

## 三、实验与结果

### 3.1 实验设置

**硬件平台**：Franka机器人

**任务类型**：
- 短时域任务 (Short-horizon)
- 长时域任务 (Long-horizon)
- 桌面操作任务

**评估指标**：任务成功率

### 3.2 主要结果

**与π₀基线对比**：

| 任务类型 | π₀基线 | Traj2Action | 提升 |
|----------|--------|-------------|------|
| 短时域任务 | 基准 | +27% | **+27%** |
| 长时域任务 | 基准 | +22.25% | **+22.25%** |

**关键发现**：
1. 3D轨迹有效弥合形态差距
2. 共去噪生成高质量动作
3. 人类数据规模增大时性能持续提升

### 3.3 消融实验

| 组件 | 贡献 |
|------|------|
| 轨迹统一表示 | 解决形态差距核心 |
| 共去噪框架 | 保持轨迹一致性 |
| 人类数据 | 提升泛化能力 |

### 3.4 与直接映射对比

| 方法 | 形态差距处理 | 动作质量 |
|------|-------------|----------|
| 直接动作映射 | 困难 | 低 |
| 关键点映射 | 部分 | 中 |
| **Traj2Action** | **轨迹统一** | **高** |

## 四、关键贡献

### 4.1 表示贡献

| 贡献 | 描述 |
|------|------|
| 3D轨迹表示 | 形态无关的统一中间表示 |
| 几何信息保留 | 保持任务的空间关系 |
| 可解释性 | 轨迹直观可理解 |

### 4.2 方法贡献

1. **轨迹专家**：联合学习人类和机器人轨迹
2. **共去噪框架**：同时处理轨迹和动作
3. **端到端学习**：无需分步骤处理
4. **形态适配**：自动适应不同机器人

### 4.3 实验贡献

- 真实世界Franka机器人验证
- 显著超越π₀基线
- 验证人类数据缩放效应

## 七、局限性与未来方向

### 7.1 局限性

1. **轨迹质量**：依赖准确的3D重建
2. **复杂操作**：多物体操作有挑战
3. **物理交互**：力信息未考虑

### 7.2 未来方向

1. 更鲁棒的轨迹提取
2. 多物体操作扩展
3. 物理感知的动作生成
4. 在线适应
5. 更多机器人形态验证
