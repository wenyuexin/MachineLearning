# Phantom：仅使用人类视频训练机器人

**论文信息**
- 论文标题：Phantom: Training Robots Without Robots Using Only Human Videos
- 中文标题：Phantom：仅使用人类视频训练机器人
- 作者：Marion Lepert, Jiaying Fang, Jeannette Bohg
- 机构：Stanford University
- 会议：CoRL 2025
- arXiv: [2503.00779](https://arxiv.org/abs/2503.00779)
- 项目主页: https://phantom-human-videos.github.io

> **核心创新**：零机器人数据训练 - 完全不需要机器人数据，仅使用人类视频即可训练可部署的机器人策略，成功率高达92%

---

## 一、论文整体思路

### 1.1 研究背景

训练通用机器人需要从大规模、多样化的数据源学习。当前方法严重依赖遥操作演示，这难以规模化。

**核心问题**：
- 机器人数据采集困难、成本高
- 人类视频数据量大但缺乏动作标签
- 人-机器人形态差距巨大

### 1.2 核心创新

Phantom提出**可扩展框架**，直接从人类视频演示训练操作策略，**完全不需要机器人数据**。

**关键突破**：
- 使用手部姿态估计和视觉数据编辑将人类演示转换为机器人兼容的观测-动作对
- 通过修复（inpaint）人类手臂并叠加渲染的机器人来对齐视觉域
- 实现零样本部署到真实硬件，无需任何微调

**性能亮点**：
- 成功率高达**92%**
- 支持可变形物体操作、多物体清扫、插入等任务
- 泛化到新场景和新物体

### 2.2 数据编辑流程

**核心步骤**：

| 步骤 | 操作 | 目的 |
|------|------|------|
| 1. 手部姿态估计 | 估计每帧中的人类手部姿态 | 提取动作信息 |
| 2. 动作转换 | 将手部姿态转换为机器人动作 | 生成可执行动作 |
| 3. 人类手臂修复 | Inpaint移除人类手臂 | 去除人类痕迹 |
| 4. 机器人叠加 | 渲染并叠加虚拟机器人 | 视觉域对齐 |
| 5. 测试对齐 | 在真实机器人观测上叠加虚拟机器人 | 确保部署一致性 |

### 2.3 机器人无关性

**核心特点**：方法与机器人无关

- 每个人类视频可转换为任意能够完成任务的机器人的演示
- 支持多种机器人部署

## 三、实验与结果

### 3.1 实验设置

**部署机器人**：
- Franka
- Kinova Gen3 (带Robotiq夹爪)

**任务类型**：
- 可变形物体操作
- 多物体清扫
- 插入任务

### 3.2 数据编辑策略对比

**三种数据编辑策略**：

| 策略 | 灵感来源 | 描述 |
|------|----------|------|
| Hand Inpaint | Rovi-Aug | Inpaint人类手部 |
| Hand Mask | Shadow | 掩盖人类手部 |
| Red Line | EgoMimic | 红线标记（已适配人类视频） |

### 3.3 主要结果

**成功率表现**：

| 环境 | Hand Inpaint | Hand Mask |
|------|--------------|-----------|
| 室外草坪 | 0.72 | 0.52 |
| 室内休息区 | 0.84 | 0.76 |
| 室内休息区+OOD表面 | 0.64 | 0.68 |

**关键发现**：
- 最高成功率：**92%**
- 支持新场景泛化
- 零样本部署到真实机器人

### 3.4 与需要机器人数据的方法对比

| 方法类型 | 机器人数据需求 | 性能 |
|----------|---------------|------|
| 传统模仿学习 | 大量 | 高（但依赖数据） |
| EgoMimic | 中等 | 高 |
| MimicPlay | 少量 | 高 |
| **Phantom** | **零** | **高达92%** |

## 四、关键贡献

### 4.1 方法论贡献

1. **零机器人数据训练**：首次实现完全无机器人数据训练
2. **数据编辑框架**：系统解决人-机器人视觉差距
3. **零样本部署**：无需微调直接部署到真实机器人

2. 普通人可用RGB-D相机收集数据

## 五、适用场景与限制

### 5.1 适用场景

1. **快速原型**：无需机器人即可测试策略
2. **数据稀缺场景**：难以获取机器人数据
3. **新任务探索**：快速验证新任务可行性
4. **多机器人部署**：同一视频可转换到多种机器人

### 5.2 局限性

1. **精细操作**：极高精度任务可能需要机器人数据
2. **形态差异大**：与人手差异大的机器人可能效果下降
3. **任务复杂度**：超长时域复杂任务仍有挑战

## 七、技术细节

### 7.1 手部姿态估计

- 使用先进的3D手部姿态估计模型
- 检测手部关键点
- 估计3D位置和姿态

### 7.2 图像修复

- Inpaint技术移除人类手臂
- 保持背景一致性
- 生成干净的训练图像

### 7.3 机器人渲染

- 渲染虚拟机器人模型
- 与修复后的图像叠加
- 保持光照和透视一致
