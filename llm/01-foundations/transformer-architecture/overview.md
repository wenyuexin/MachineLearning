# LLM 架构技术总览

## 一、核心架构组件

### 1.1 Transformer 基础架构

标准 Transformer 由多头注意力和前馈网络交替堆叠构成。当前 LLM 架构改进主要围绕以下方向：

| 组件 | 核心问题 | 改进方向 |
|------|----------|----------|
| **注意力** | O(n^2) 复杂度 | 稀疏注意力、线性注意力、分组查询注意力 |
| **位置编码** | 外推性差 | RoPE、ALiBi、动态位置编码 |
| **前馈网络** | 参数量大 | MoE、专家混合 |
| **归一化** | 训练稳定性 | Pre-Norm (RMSNorm) |

### 1.2 注意力机制

```
标准多头注意力 (MHA)
    ↓ 演进
分组查询注意力 (GQA)  ← KV头共享，平衡效率和质量
    ↓ 演进
多查询注意力 (MQA)    ← 所有KV头共享，最高效
```

**高效注意力方法**：稀疏注意力、线性注意力、FlashAttention、滑动窗口注意力

详见 → [attention/](./attention/)

### 1.3 位置编码

| 方法 | 类型 | 外推能力 | 代表模型 |
|------|------|----------|----------|
| RoPE | 相对位置 | 中等（可扩展） | LLaMA, Qwen |
| ALiBi | 线性偏置 | 较好 | BLOOM |
| YaRN | RoPE缩放 | 好 | — |
| NTK-aware | RoPE缩放 | 较好 | Code LLaMA |

详见 → [position-encoding/](./position-encoding/)

## 二、架构范式

### 2.1 Dense Transformer

标准稠密 Transformer，所有参数在每次推理中全部激活。

- 代表：GPT-4, LLaMA 3, Qwen 2.5
- 优势：训练简单、推理稳定
- 劣势：参数效率低、扩展成本高

### 2.2 混合专家 (MoE)

稀疏激活架构，每次推理只激活部分专家。

```
输入 → [路由器] → 选择Top-K专家 → [专家计算] → 合并输出
```

- 代表：Mixtral, DeepSeek-V2/V3/V4, Qwen-MoE
- 优势：参数量大但计算量可控
- 劣势：负载均衡、训练复杂

详见 → [moe/](./moe/)

### 2.3 状态空间模型 (SSM)

线性复杂度的替代架构，兼具并行训练和高效推理。

| 模型 | 核心机制 | 复杂度 | 长上下文能力 |
|------|----------|--------|-------------|
| Mamba | 选择性状态空间 | O(n) | 好 |
| RWKV | 线性注意力+RNN | O(n) | 好 |
| RetNet | 多尺度保留 | O(n) | 好 |
| Jamba | Mamba+Attention混合 | O(n)~O(n^2) | 兼顾 |

详见 → [ssm/](./ssm/)

## 三、长上下文

长上下文是当前 LLM 架构研究的核心方向之一。详见专题 → [long-context/](./long-context/)

**关键技术路线**：
- 原生长上下文架构（Gemini 1.5, MiniMax M1）
- 后期上下文扩展（RoPE缩放、位置内插）
- 注意力机制优化（稀疏、滑动窗口）
- 推理效率优化（KV缓存压缩、量化、投机解码）

---

## 四、架构发展趋势

1. **MoE成为主流**：从Dense到MoE，参数效率持续提升
2. **混合架构兴起**：Attention+SSM的混合成为新方向（Jamba等）
3. **长上下文标配化**：1M上下文窗口成为旗舰模型标配
4. **推理优化关键化**：随着模型规模和上下文长度增长，推理优化成为核心瓶颈
5. **架构搜索自动化**：从手工设计到自动搜索更优架构
