## 核心问题

长上下文是LLM的核心竞争能力。当前上下文长度已从4K扩展到1M+，但仍面临三大挑战：

1. **架构瓶颈**：注意力机制O(n^2)复杂度，位置编码外推性差
2. **推理效率**：KV缓存显存爆炸，自回归解码延迟高
3. **评估难题**：如何区分检索能力和真正的长上下文理解

## 技术路线

| 路线 | 描述 | 代表 |
|------|------|------|
| **原生架构** | 训练时即支持长上下文 | Gemini 1.5 Pro, MiniMax M1 |
| **后期扩展** | 短上下文模型扩展到长上下文 | Qwen2.5-1M, LWM |
| **位置编码调整** | RoPE缩放、ALiBi等 | NTK-aware, YaRN |
| **注意力优化** | 稀疏注意力、线性注意力 | Longformer, Mamba |
| **推理优化** | 量化、KV缓存压缩、投机解码 | FlashAttention, PagedAttention |
