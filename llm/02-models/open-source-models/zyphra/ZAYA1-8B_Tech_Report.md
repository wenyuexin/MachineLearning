## 一、论文整体思路

### 1.1 研究背景：推理能力与效率的双重挑战

推理模型（Reasoning Model）的出现构建了测试时间缩放（Test-Time Scaling）的全新范式，为大语言模型带来了显著的性能提升。然而，这种范式在实践中面临一个根本性矛盾：**推理能力通常以参数规模的扩张为代价**——更大的模型带来更强的推理性能，但同时也带来了更高的推理成本和部署门槛。

与此同时，开源社区在提升通用能力方面取得了长足进步，但在"以小博大"这一方向上——即**以极低的激活参数预算实现可比的推理性能**——仍缺少系统性探索。如何在模型架构、训练策略和推理算法三个层面协同设计，使得小型模型在推理密集型任务上达到前沿水平，成为一项关键课题。

### 1.2 ZAYA1-8B的定位与核心贡献

ZAYA1-8B的定位非常明确：**在极低的激活参数预算下，通过全栈协同设计实现超越同规模乃至更大规模模型的推理性能**。模型基于Zyphra自研的MoE++架构，核心规格如下：

| 属性 | 配置 |
|------|------|
| 架构 | Decoder-only MoE Transformer（MoE++） |
| 总参数 | **8.4B** |
| 活跃参数 | **0.76B（760M）** |
| 层数 | 40层 |
| 隐藏维度 | 2048 |
| 注意力变体 | CCGQA + CCA预处理器 |
| KV-cache压缩比 | **8x**（相对全多头注意力） |
| 专家数/每MoE层 | 16 |
| 路由方式 | Top-1，无残差专家 |
| 位置编码 | 每头50%维度RoPE |
| 分词器 | Gemma3分词器，262,272词表 |

核心贡献可归纳为三大架构创新与两大训练/推理突破：

1. **压缩卷积注意力（CCA）**：在压缩潜在空间内完成序列混合，实现8倍KV-cache压缩
2. **MLP专家路由器（ZAYA1 Router）**：以多层MLP替代标准线性路由器，提升路由表达力和稳定性
3. **可学习残差缩放（Learnable Residual Scaling）**：以极小的参数和FLOP开销控制深层网络残差范数增长
4. **四阶段RL级联**：从推理热身到行为对齐的渐进式后训练流程
5. **马尔可夫RSA**：一种新颖的测试时计算方法，以固定内存成本实现近乎无限的迭代推理

在关键成果上，ZAYA1-8B以**760M活跃参数**在AIME'26上达到**89.1**，超越Mistral-Small-4-119B（86.4）和Qwen3-4B（77.5）；运用马尔可夫RSA后，在AIME'25上达到**91.9%**，在HMMT'25上达到**89.6%**，缩小了与Gemini-2.5 Pro、DeepSeek-V3.2和GPT-5-High等更大推理模型的差距。

## 二、模型架构总览

### 2.1 架构设计哲学：以小博大的"智能密度"

ZAYA1-8B的设计围绕一个核心理念展开：**最大化单位参数和单位FLOP产生的"智能密度"（Intelligence Density per Parameter）** 。Zyphra在报告中指出："效率一直是Zyphra的核心指导原则，影响着我们如何设计模型架构、开发训练与推理算法，并选择具备最佳性价比的硬件。"

这一哲学具体体现在：

```
传统范式：    增大模型参数 → 提升推理能力 → 推理成本线性增长
ZAYA1-8B路线：精妙架构设计 → 提升单位参数效率 → 推理成本亚线性增长
                + 针对性训练策略
                + 高效的测试时计算
```

## 三、模型架构详解

### 3.1 压缩卷积注意力（CCA）

CCA是Zyphra开发的一种FLOP和内存高效的注意力变体，其核心理念是：**将查询（Q）、键（K）和值（V）通过轻量级卷积下投影器映射到更低的潜在空间，然后在该共享潜在空间内完成全部注意力计算**。

#### 3.1.1 计算流程

标准多头注意力的计算流程为：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

CCA在此基础上引入压缩变换：

1. **卷积下投影**：Q、K、V分别通过一个轻量级卷积下投影器（Convolutional Downprojector），将维度从 $d_{\text{model}}$ 压缩到 $d_{\text{latent}}$
2. **潜在空间注意力**：在压缩后的潜在空间内执行多头注意力计算（CCGQA：8查询头，2KV头）
3. **上投影还原**：将注意力输出上投影回原始维度

#### 3.1.2 三重补偿机制

为保障压缩后的性能，CCA引入了三重补偿机制：

| 补偿机制 | 作用 | 实现方式 |
|---------|------|----------|
| **卷积混合** | 在压缩后的Q和K潜变量上逐序列和逐通道应用卷积 | 增强局部上下文感知能力 |
| **QK均值自适应** | 在卷积前后对Q和K值取均值 | 稳定压缩空间内的注意力分布 |
| **值偏移** | 每个注意力头同时接收当前和前一个位置的token嵌入 | 补偿压缩导致的位置信息损失 |

#### 3.1.3 效率优势

CCA的核心效率优势体现在：

| 维度 | 标准MHA | CCA（ZAYA1-8B） | 收益 |
|------|---------|-----------------|------|
| KV-cache大小 | 100%（基线） | **12.5%** | 8倍压缩 |
| Query压缩 | 无 | 2x | 减少预填充计算 |
| 长上下文推理 | 内存随序列线性增长 | 内存增长大幅减缓 | 支持更高效的推理 |

CCA的查询压缩率为2倍，KV压缩率为8倍（相对全多头注意力），有效降低了训练和预填充阶段的计算需求，同时大幅削减了长上下文解码时的KV-cache体积。

#### 3.1.4 与其他注意力机制的对比

报告将CCA与MLA（多头潜在注意力，DeepSeek）和GQA（分组查询注意力）进行了对比，指出CCA在保持竞争力的同时，在KV-cache压缩比和计算效率上具有独特优势。ZAYA1-8B的推理和长上下文性能证明了CCA在此规模下依然有效，能够支撑推理、上下文学习和长距离召回等任务。

### 3.2 MLP基专家路由器（ZAYA1 Router）

标准MoE模型通常采用**线性投影**来确定token由哪个专家处理：

$$p = \text{softmax}(W_{\text{routing}} \cdot x)$$

ZAYA1-8B则采用**多层感知机（MLP）**作为路由器，在标准线性投影前引入非线性变换：

$$p = \text{softmax}(\text{MLP}(x))$$

路由器的关键规格：

| 属性 | 配置 |
|------|------|
| 架构 | MLP（多层） |
| 潜在维度 | 256 |
| 路由方式 | Top-1（每token激活1个专家） |
| 偏差平衡 | PID控制器风格 |
| 专家数/层 | 16 |

#### 3.2.1 MLP路由器的优势

相比标准线性路由器，MLP路由器提供了**更强的表达能力**：

- **更一致的路由模式**：实验表明，MLP路由器的专家激活模式更加稳定，token-to-expert的分配更加明确
- **更高的路由置信度**：路由分配的概率分布更加尖锐，表明模型对路由决策更加确信
- **更好的专家专业化**：MLP路由器促进了专家的深层专业化分工

作为专家冗余度的合理性检查，论文附录D测量了每层内的专家子空间重叠（Expert Subspace Overlap）：

| 模型 | 一阶投影输入重叠（相对随机基线） |
|------|:-------------------------------:|
| ZAYA1-8B | **1.45×** |
| Qwen3-30B-A3B | 1.48× |
| 其他MoE基线 | 中等范围 |

ZAYA1-8B的专家重叠度并非离群值——其值与Qwen3-30B-A3B接近，输出投影重叠在对比模型中处于中等水平，说明MLP路由器虽然增加了路由器容量，但没有导致专家冗余的异常增加。

#### 3.2.2 PID控制器风格偏差平衡

为主动防止专家负载不均，ZAYA1-8B引入了PID控制器风格的偏差平衡机制。该方法借鉴了经典控制理论的思路——通过比例（Proportional）、积分（Integral）和微分（Derivative）三项误差信号的加权组合来动态调整路由偏差，使得各专家的负载保持均衡。

### 3.3 可学习残差缩放

随着模型深度增加，残差范数会逐层不稳定增长。ZAYA1-8B的**可学习残差缩放**（Learnable Residual Scaling）以极小的参数和FLOP开销，动态调整残差连接的大小：

$$y = \alpha \cdot x + \beta \cdot f(x)$$

其中 $\alpha, \beta$ 为可学习的标量系数（每层独立）。这种方法确保信号在深层网络中的稳定传播，从而稳定深度MoE配置（40层）的训练。

关键特点：
- **参数开销极小**：每层仅增加2个标量参数
- **FLOP开销可忽略**：几乎不影响前向计算速度
- **训练效果显著**：消融实验表明，移除该机制导致AIME准确率下降5.2%

Zyphra同时指出，残差缩放机制可能有助于保留深层网络对最终输出的贡献——随着深度增加，残差范数的可控增长确保了层间信号的均衡传递。

### 3.4 位置编码

ZAYA1-8B在位置编码上采用了一项特殊设计：**在每个注意力头上，仅对50%的通道施加RoPE位置编码**，剩余50%的通道不编码。这一设计在保留位置感知能力和避免RoPE对压缩KV空间的不一致影响之间取得了平衡。

分词器方面，模型采用了Gemma3分词器，词表大小为262,272 tokens。

## 四、训练策略

### 4.1 推理优先的预训练与保留答案的修剪

ZAYA1-8B从预训练阶段就为推理任务而设计。模型的总预训练数据量约为**12万亿tokens**（报告未披露精确的混合比例），数据涵盖通用语料、数学和代码内容。

团队采用了一种名为**"保留答案的修剪"（Answer-Preserving Trimming）** 的方案：在处理长推理数据时，保留那些导向正确答案的推理链条，同时截断冗余或无关的部分。使推理数据从一开始就融入模型权重，而非仅在后期训练阶段附加。

回答保留修剪的核心理念：
- **长推理链包含大量步骤**，但并非所有步骤都对最终答案有贡献
- **修剪冗余步骤**，保留关键推理路径
- **训练信号更密集**：每个训练样本都聚焦于有信息量的推理步骤

### 4.2 中期训练与监督微调

在预训练之后，ZAYA1-8B经历了中期训练（Midtraining）和监督微调（SFT）阶段：

- **中期训练**：在长上下文数据上进一步训练，为后续RL阶段的推理任务准备基础能力。CCA的低计算和通信成本在此阶段发挥了重要作用
- **监督微调**：在高质量推理数据上进行SFT，生成初始的SFT检查点。该检查点在AIME'26上得分为68.30，HMMT'26 Feb上得分为39.20

SFT还引入了马尔可夫RSA所需的**基于聚合的训练样本**（Aggregation-based Examples）。这些样本的构造方式为：向模型呈现一个问题及若干候选推理尾部（reasoning tails），然后训练模型基于这些尾部生成一个改进后的答案。这为后续RL阶段的测试时计算训练奠定了基础。

### 4.3 长度奖励机制

为鼓励模型生成简洁高效的推理链，ZAYA1-8B设计了一套**难度感知的长度奖励**（Difficulty-Scaled Length Reward），综合借鉴了ALP和ShortRL的思想。

给定一个prompt的rollout组大小 $G$、响应奖励 $r_i \in \{0,1\}$ 和响应长度 $\ell_i$，首先计算组求解率 $p = \frac{1}{G}\sum_{i=1}^{G} r_i$ 和最短正确响应长度 $\ell_{\min}$。长度插值定义为：

$$\lambda_i = \frac{1}{2} - \text{clamp}\!\left(\frac{\ell_i - \ell_{\min}}{\ell_{\max} - \ell_{\min}},\;0,\;1\right) \tag{8}$$

其中 $\ell_{\max}$ 为常数。该插值将正确响应按长度映射到 $[-0.5, 0.5]$ 区间——短于 $\ell_{\min}$ 的响应获得正奖励，长于 $\ell_{\max}$ 的获得负奖励。

为只在有意义的比较场景下激活长度奖励，设计了一个正确性和难度门控掩码：

$m_i = \mathbb{1}\!\left[p > p^{*} - T_{\text{acc}}\right] \cdot \mathbb{1}\!\left[p > \tfrac{1}{G}\right] \cdot \mathbb{1}\!\left[r_i = 1\right] \tag{9}$

其中：
- $p^{*}$ 为对应数据源或环境的**运行最大求解率**（Running Maximum Solve Rate）
- $T_{\text{acc}}$ 为容忍阈值，防止长度奖励在远低于当前能力边界的任务上激活
- 条件 $p > 1/G$ 等价于 $k \geq 2$（组内至少有2个正确回答时才存在有意义的长度比较）

最终的长度奖励为：

$\Delta r_i = c \cdot p \cdot m_i \cdot \tilde{\lambda}_i \tag{10}$

其中 $c$ 为缩放系数，$\tilde{\lambda}_i$ 在 $\ell_i \leq \ell_{\min} + T_{\ell}$ 时固定为 $0.5$（保护最短正确答案免受惩罚），其余情况等于 $\lambda_i$。

**关键设计特征**：
- 难度感知：奖励乘以求解率 $p$，在困难问题上衰减长度惩罚，在简单问题上放大
- 渐进激活：长度奖励系数 $c$ 从推理热身阶段的小值逐渐提升，到数学+代码+TTC阶段达到 $c=1.0$
- 仅在优势函数分子中生效：$\hat{A}_i = \frac{r_i + \Delta r_i - \bar{r}}{\bar{r}}$，分母 $\bar{r}$ 使用未修改的任务奖励以避免尺度膨胀

### 4.4 四阶段强化学习级联

ZAYA1-8B的后训练采用了一个循序渐进的**四阶段RL级联**流程（Post-training RL Cascade），每个阶段的目标明确、层层递进。整个级联共享统一的算法框架：

**统一算法框架**：
- **异步PipelineRL**：使用异步流水线RL框架，支持多阶段RL的高效调度
- **DPPO Binary-TV信任域掩码**（Binary Total-Variation Trust-Region Masking）：在策略更新时施加信任域约束，防止单步更新过大导致训练不稳定
- **Dr-GRPO序列级损失聚合**（Sequence-Level Loss Aggregation）：在序列级别聚合GRPO损失，而非逐token级别，简化梯度计算
- **MaxRL优势估计**：使用方差缩减的MaxRL估计器
  $\hat{A}_i = \frac{r_i - \bar{r}}{\bar{r}}, \quad \bar{r} = \frac{1}{G}\sum_{j=1}^{G} r_j \tag{7}$
  相比标准GRPO，该估计器对困难prompt产生更强的梯度信号
- **奖励中不使用KL正则化**：不同于标准PPO/GRPO中将KL散度作为惩罚项，ZAYA1-8B的奖励计算中不包含KL项

#### 4.4.1 RLVE-Gym：自适应课程学习

RLVE-Gym是ZAYA1-8B训练中最具创新性的组件之一。它是一个包含**400个可验证环境**（Verifiable Environments）的自适应课程系统，源自Zeng et al. (2025a)。

RLVE-Gym的核心是一个**难度参数化调度器**。每个环境 $m$ 关联一组难度参数 $(\mu_m, s_m)$，构成参数池 $\Theta$：

$\Theta = \{(\mu_m, s_m)\}_{m=1}^{M}$

给定一个难度 $d$，模型在该环境上的预期求解概率建模为：

$p_{\text{success}} = \sigma\!\left(-\frac{d - \mu}{s}\right) = \frac{1}{1 + e^{(d - \mu)/s}} \tag{12}$

其中 $\mu$ 控制难度中位点（$p_{\text{success}}=0.5$ 时的 $d$ 值），$s$ 控制难度过渡带的宽度。

**自适应调度流程**：
1. 初始化参数池中所有候选的权重为均匀分布
2. 每次迭代按后验权重从 $\Theta$ 中采样一组参数
3. 使用采样参数计算目标难度 $d$ 处的 $p_{\text{target}}$
4. 根据模型在该难度下的实际表现更新后验权重
5. 自动聚焦于模型"适度挑战区"（既不过难也不过易的环境）

- **动态难度调整**：每个环境根据模型实时成功率自动调节难度
- **适度挑战区**：使模型始终处于"不过难也不过易"的训练区间
- **可验证奖励**：每个环境的正确性可以通过自动化验证（如数学答案验证、代码测试通过）

RLVE-Gym的设计体现了"课程学习"的先进理念，确保模型系统性地获得广泛且深入的泛化推理能力。

#### 4.4.2 第三阶段：数学与代码强化学习

## 五、测试时计算：马尔可夫RSA

**马尔可夫RSA**（Markovian RSA）是论文中最具原创性的贡献之一——一种创新的测试时计算方法，其核心思想是：**并行生成多条独立的、非常长的推理链，然后通过递归聚合的方式，只将固定长度的"尾部"信息向前传递**。

### 5.1 设计动机

传统测试时计算面临的核心困境：生成长推理链虽然能提升准确性，但上下文窗口持续增长，内存和计算成本线性膨胀。马尔可夫RSA通过**马尔可夫式的"有界工作空间"** 解决了这一问题——模型每次只关注推理历史的固定长度尾部，而非完整历史。

### 5.2 核心算法

马尔可夫RSA的工作流程如下：

```
初始提示
    ↓
┌────────────────────────────────────────────┐
│  并行推理阶段                              │
│  ┌─────┐ ┌─────┐         ┌─────┐          │
│  │链 1 │ │链 2 │   ···   │链 N │          │
│  └──┬──┘ └──┬──┘         └──┬──┘          │
│     ↓       ↓               ↓             │
│  ┌──────────────────────────────┐         │
│  │ 各链尾部截取（固定4K tokens） │          │
│  └──────────────┬───────────────┘         │
└─────────────────┼─────────────────────────┘
                  ↓
┌────────────────────────────────────────────┐
│  递归聚合阶段                              │
│  ┌────────────────────────────────────┐   │
│  │ 将各链尾部信息递归聚合              │   │
│  │ → 只携带有限的推理历史进入下一轮    │   │
│  └────────────────────────────────────┘   │
                  ↓
           思考-截取-再思考循环
```

**算法定义**：给定一个问题 $q$ 和基础策略 $\pi$，马尔可夫RSA在 $T$ 轮聚合轮次（Aggregation Rounds）中进行，索引为 $t = 0, 1, \ldots, T$。每轮维护 $N$ 条候选推理链（Candidate Reasoning Traces）。关键参数如下：

| 参数 | 含义 | 默认配置 |
|------|------|----------|
| $T$ | 聚合轮数 | 2 |
| $N$ | 每轮并行生成的候选数 | 16 |
| $\beta$ | 每候选单轮推理预算（token数） | 8K / 16K / 40K |
| $\tau$ | 携带的尾部长度（token数） | 4K |
| $C$ | 向前传递的候选尾部数 | 4 |

**具体实现**：

1. **初始化（$t=0$）**：模型直接从问题 $q$ 生成 $N$ 条独立rollout，每条有推理预算 $\beta$。每条推理链截取最后 $\tau$ 个token作为"尾部"（Tail），记作 $\text{tail}_{\tau}(y)$，其中 $\tau \leq \beta$
2. **聚合（$t > 0$）**：将上一轮 $C$ 条候选的尾部拼接为聚合prompt，模型基于该prompt再次生成 $N$ 条新推理链
3. **尾部截取**：每次生成后仅保留最后的 $\tau$ token向前传递
4. **循环迭代**：重复聚合-生成-截取流程，直到达到预设的聚合轮数 $T$ 或满足终止条件

### 5.3 核心优势

| 维度 | 传统TTC | 马尔可夫RSA |
|------|---------|------------|
| 上下文窗口 | 线性增长 | **恒定（4K token）** |
| 内存成本 | 随推理深度线性增加 | **恒定** |
| 推理深度 | 受限于序列长度 | **近乎无限** |
| 并行效率 | 单链串行 | **多链并行（可批量处理）** |
| 适用场景 | 短推理链 | 长链/迭代推理 |

### 5.4 性能飞跃

在资源最密集的配置下，ZAYA1-8B以**每问题550万token**的计算预算，在APEX-shortlist数学基准上超越了DeepSeek-V3.2和GPT-OSS-High。

在TTC评估中，马尔可夫RSA的表现尤为亮眼。不同资源配置下的性能对比：

| 配置 | $T$ | $N$ | $C$ | $\beta$（推理预算） | $\tau$（尾部长度） | **AIME'25** | **HMMT'25** |
|:---|:---:|:---:|:---:|:----------------:|:----------------:|:---------:|:---------:|
| 马尔可夫基线 | 4 | 16 | 1 | 8K | 4K | 82.1 | 75.0 |
| 马尔可夫RSA | 2 | 16 | 4 | 8K | 4K | 86.5 | 80.8 |
| 马尔可夫RSA | 2 | 16 | 4 | 16K | 4K | 88.8 | 87.1 |
| **马尔可夫RSA** | **2** | **16** | **4** | **40K** | **4K** | **91.9** | **89.6** |
| 马尔可夫RSA | 2 | 16 | 4 | 40K | 8K | 90.8 | 89.2 |

关键发现：
- **$\beta$（推理预算）是核心杠杆**：从8K提升到40K，AIME'25提升5.4个百分点（86.5→91.9），HMMT'25提升8.8个百分点（80.8→89.6）
- **尾部长度$\tau$的影响**：4K和8K尾部在40K预算下性能接近（91.9 vs 90.8），说明4K尾部已足够捕捉关键推理信息
- **聚合($C>1$)优于无聚合($C=1$)**：马尔可夫基线($C=1$)显著低于聚合配置，验证了跨候选信息融合的价值

即使仅维持4K token的推理尾部，模型就缩小了与Gemini-2.5 Pro、DeepSeek-V3.2、Qwen3-235B-A22B-Thinking-2507和GPT-5-High等更大推理模型的差距。

### 5.5 与Full-Chain RSA的对比

马尔可夫RSA与原始的**全链RSA**（Full-Chain RSA，Venkatraman et al., 2025）在聚合方式上存在关键区别：

| 对比维度 | Full-Chain RSA | 马尔可夫RSA |
|---------|---------------|------------|
| 聚合上下文 | 完整推理历史 | 仅尾部（$\tau$ tokens） |
| 每聚合项token数（$\beta=40K, C=4$） | 最多**160K tokens** | 仅**16K tokens** |
| 内存成本 | 随推理深度线性增长 | 恒定 |
| 适用场景 | 小规模/短链推理 | 大规模/迭代推理 |

在 $\tau = \beta$ 的极限条件下，马尔可夫RSA退化为Full-Chain RSA。但$\tau=4K, \beta=40K, C=4$的设置下，每聚合项的token数从160K锐减至16K——**90%的上下文缩减**。因此论文使用马尔可夫RSA进行高预算评估，而将Full-Chain RSA的全面对比留给未来工作。

### 5.6 训练与推理的协同设计

马尔可夫RSA的一个重要设计特征是：**测试时计算方法与训练过程协同设计**。在SFT和RL阶段，模型已经接触过类似的推理痕迹模式——模型在训练中见过"思考-截取-聚合"的流程，因此在推理时能够更有效地利用这一策略。这与事后附加的测试时计算方法形成鲜明对比。

## 八、总结与技术展望

### 8.1 核心贡献回顾

ZAYA1-8B的成功可以归结为三个层面的协同创新：

1. **架构层面**：
   - **CCA**：以8倍KV-cache压缩比显著降低长上下文推理成本
   - **MLP路由器**：提升路由稳定性和专家专业化程度
   - **可学习残差缩放**：以可忽略的额外成本稳定40层深度MoE训练

2. **训练层面**：
   - **推理优先的预训练**：通过保留答案的修剪，从源头植入推理能力
   - **四阶段RL级联**：从热身到对齐的渐进式流程，RLVE-Gym以动态难度系统性地培养泛化推理
   - **SFT→RL的巨大提升**：AIME提升20.8分，HMMT提升32.4分

3. **推理层面**：
   - **马尔可夫RSA**：以固定4K token内存成本换取近乎无限的测试时思考时间
   - 以每问题550万token的计算预算在APEX-shortlist上超越DeepSeek-V3.2和GPT-OSS-High

### 8.2 技术启示

- **用小参数做大推理**：ZAYA1-8B证明，竞争性数学推理能力可以在低于1B活跃参数的条件下实现，当模型架构、训练策略和测试时计算经过协同设计时
- **深度可能比宽度更重要**：40层深度（对比同规模的16-36层）为模型提供了更多串行计算，可能对推理产生重要贡献
- **可验证RL是关键**：数学和代码领域的可验证反馈信号是RL训练有效性的核心驱动力
- **测试时计算与训练共设计**：将TTC方法融入训练过程（而非事后附加）显著提升了推理时的TTC利用效率
- **统一的RL算法框架偏好**：整个RL级联共享统一的DPPO Binary-TV + Dr-GRPO + MaxRL框架，无需为不同阶段切换算法，降低了工程复杂度

### 8.3 开源与生态

ZAYA1-8B已在**Apache 2.0协议下开源**，提供[Hugging Face](https://huggingface.co/Zyphra/ZAYA1-8B)模型权重和Zyphra Cloud服务端点。这一开源策略为开发者在成本与性能之间提供了一个全新选择。

Zyphra同期还发布了**ZAYA1-74B-Preview**预览版（74B总参数，4B活跃参数），展示了同一条技术路线在大规模模型上的延续潜力。

### 8.4 开放问题与未来方向

报告在结论部分提出了若干有待进一步探索的问题：

1. **深度假设的因果验证**：40层深度对推理能力的贡献仍需更严格的控制消融实验来验证
2. **Agent和事实性基准的差距**：尽管在数学和编程上表现出色，但在智能体任务和事实性知识检索上仍有提升空间
3. **测试时计算的收益边界**：马尔可夫RSA的收益是否存在上限？增加并行链数量 $N$、聚合轮数 $T$ 和迭代轮次的边际收益递减规律如何？
4. **Full-Chain RSA vs 马尔可夫RSA的全面对比**：论文将全面对比留给未来工作，在更大规模和更多基准上的经验比较仍有待探索
5. **路由器的专家重叠分析**：报告的附录D分析表明ZAYA1-8B的专家子空间重叠处于正常范围，但对路由器行为的深层理解——如MLP路由器在更深层网络中的行为变化——仍有待推进

展望未来，密集模型仍是许多应用的主流选择，但ZAYA1-8B已清晰地描绘出一种新的可能性：**低推理成本不一定以高训练开销为代价，算法与硬件的协同优化始终是突破算力瓶颈的核心路径**。
