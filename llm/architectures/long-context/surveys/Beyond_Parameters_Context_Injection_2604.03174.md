## 一、论文整体思路

### 1.1 研究背景

大语言模型在参数中编码了大量世界知识，但它们本质上受限于静态知识、有限的上下文窗口和弱结构化的因果推理能力。上下文增强（Contextual Enrichment）——在推理时提供结构化上下文——成为弥补这些局限的关键策略。

### 1.2 综述定位与贡献

本综述沿单一轴线组织增强策略：推理时提供的结构化上下文程度，主要贡献：

1. **统一视角**：从上下文学习到因果RAG的统一分类
2. **文献筛选协议**：透明的论文筛选方法
3. **声明审计框架**：区分高置信度和新兴发现
4. **部署导向**：提供面向实际部署的决策框架

## 二、上下文增强方法谱系

### 2.1 方法谱系

```
上下文程度: 低 ←────────────────────────────────→ 高

上下文学习     提示工程      RAG      GraphRAG    CausalRAG
(ICL)         (Prompting)           (图增强)    (因果增强)
```

### 2.2 方法对比

| 方法 | 上下文程度 | 知识来源 | 推理能力 | 实现复杂度 |
|------|-----------|----------|----------|-----------|
| **上下文学习 (ICL)** | 低 | 模型参数+少样本 | 依赖模型 | 低 |
| **提示工程** | 中低 | 模型参数+指令 | 依赖模型 | 低 |
| **RAG** | 中 | 外部文档+模型 | 受限于检索 | 中 |
| **GraphRAG** | 中高 | 知识图谱+模型 | 结构化推理 | 高 |
| **CausalRAG** | 高 | 因果图+模型 | 因果推理 | 很高 |

## 三、上下文学习与提示工程

### 3.1 上下文学习 (ICL)

| 维度 | 描述 | 关键发现 |
|------|------|----------|
| **示例选择** | 如何选择ICL示例 | 示例相关性比多样性更重要 |
| **示例排列** | 示例的顺序影响 | 近因效应显著 |
| **示例数量** | 多少示例最优 | 边际效益递减 |
| **格式设计** | 示例的格式化方式 | 一致的格式提升性能 |

### 3.2 提示工程

| 技术 | 描述 | 适用场景 |
|------|------|----------|
| **Chain-of-Thought** | 逐步推理提示 | 复杂推理 |
| **Self-Consistency** | 多路径推理+投票 | 提高可靠性 |
| **Tree-of-Thought** | 树状推理结构 | 探索性问题 |
| **ReAct** | 推理+行动交织 | 工具使用 |

## 四、检索增强生成 (RAG)

### 4.1 RAG架构

```
查询 → [检索器] → 检索文档
                        ↓
查询 + 检索文档 → [生成器/LLM] → 生成回答
```

### 4.2 RAG分类

| 类别 | 描述 | 代表方法 |
|------|------|----------|
| **朴素RAG** | 检索-读取 | 原始RAG |
| **高级RAG** | 检索前后优化 | 查询改写、重排序 |
| **模块化RAG** | 灵活模块组合 | Self-RAG, CRAG |
| **迭代RAG** | 多轮检索精炼 | ITER-RETGEN |

### 4.3 RAG的关键挑战

| 挑战 | 描述 | 解决方向 |
|------|------|----------|
| **检索质量** | 检索到的文档不相关 | 更好的检索器、重排序 |
| **上下文窗口限制** | 检索文档过长 | 压缩、选择 |
| **幻觉** | 模型忽略检索内容 | 忠实度约束 |
| **延迟** | 检索增加推理延迟 | 缓存、预取 |

## 五、GraphRAG

### 5.1 GraphRAG架构

```
文档 → [实体抽取] → 知识图谱
                         ↓
查询 → [图检索] → 子图 → [LLM] → 生成回答
```

### 5.2 GraphRAG vs RAG

| 维度 | RAG | GraphRAG |
|------|-----|----------|
| **知识表示** | 文本片段 | 结构化图 |
| **检索方式** | 向量相似度 | 图遍历 |
| **推理能力** | 文本推理 | 结构化推理 |
| **多跳能力** | 受限 | 原生支持 |
| **构建成本** | 低 | 高 |
| **适用场景** | 事实问答 | 关系推理 |

---

## 六、CausalRAG

### 6.1 因果增强

**核心思想**：将因果推理引入检索增强生成，使模型不仅能检索相关事实，还能理解因果关系。

| 维度 | 传统RAG | CausalRAG |
|------|---------|-----------|
| **关联vs因果** | 检索相关性 | 识别因果链 |
| **推理类型** | 关联推理 | 因果推理 |
| **反事实** | 不支持 | 支持 |
| **可解释性** | 低 | 高 |

### 6.2 CausalRAG的方法

1. **因果图构建**：从文档中抽取因果关系构建因果图
2. **因果检索**：基于因果路径而非相关性的检索
3. **因果推理**：沿因果链进行推理
4. **反事实推理**：支持"如果...会怎样"的推理

## 七、部署决策框架

### 7.1 方法选择指南

| 场景 | 推荐方法 | 理由 |
|------|----------|------|
| 简单问答 | ICL + 提示 | 成本低，效果够 |
| 知识密集型问答 | RAG | 需要外部知识 |
| 关系推理 | GraphRAG | 需要结构化知识 |
| 因果分析 | CausalRAG | 需要因果推理 |
| 实时部署 | 缓存+RAG | 平衡延迟和准确性 |
