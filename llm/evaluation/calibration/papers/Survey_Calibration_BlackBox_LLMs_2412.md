## 一、论文整体思路

### 1.1 研究背景

随着 GPT-4、Claude 等闭源 LLM 的普及，黑盒场景（仅能通过 API 访问模型输出）成为主流部署方式。传统校准方法大多假设白盒访问（可获取 logits、隐藏状态），在黑盒场景中无法使用。

### 1.2 核心问题

**如何在没有模型内部访问权限的情况下，评估和改善 LLM 的校准质量？**

### 1.3 主要贡献

1. **系统梳理黑盒校准方法**：覆盖置信度引出、校准评估、校准改善三个环节
2. **方法分类框架**：按信息获取方式对黑盒校准方法进行系统分类
3. **挑战与展望**：总结黑盒校准的独特挑战和未来方向

## 二、黑盒校准的方法分类

### 2.1 校准流程三环节

```mermaid
flowchart LR
    A[置信度引出<br/>Confidence Elicitation] --> B[校准评估<br/>Calibration Evaluation]
    B --> C[校准改善<br/>Calibration Improvement]
```

| 环节 | 白盒方法 | 黑盒方法 |
|------|---------|---------|
| **置信度引出** | 读取 logits/softmax | Prompt 引出语言化置信度 |
| **校准评估** | 直接计算 ECE/Brier | 基于引出置信度计算，或基于一致性间接评估 |
| **校准改善** | Temperature Scaling 等 | Prompt 工程、Few-shot 校准示例 |

### 2.2 置信度引出方法

#### 基于多次采样的方法

| 方法 | 原理 | 信息需求 | 成本 |
|------|------|---------|------|
| **Self-Consistency** | 多次采样答案一致性 | 仅文本输出 | 高（需多次生成） |
| **语义熵** | 语义聚类后的熵 | 文本输出 + NLI模型 | 高 |
| **图度量（EigV/Deg/Ecc）** | 语义相似度图 | 文本输出 + NLI模型 | 中 |

#### 基于语言化表达的方法

| 方法 | 原理 | 信息需求 | 成本 |
|------|------|---------|------|
| **Verbalized Confidence** | 直接询问置信度百分比 | 单次生成 | 低 |
| **p(True)** | 让模型判断答案正确性 | 单次生成 | 低 |
| **语言概率表达** | 选择语言置信度等级 | 单次生成 | 低 |

### 2.3 黑盒 vs 白盒方法对比

| 维度 | 白盒方法 | 黑盒方法 |
|------|---------|---------|
| 访问需求 | logits、隐藏状态 | 仅文本输出 |
| 代表方法 | Token熵、Temperature Scaling | 语义熵、Self-Consistency、Verbalized |
| 计算成本 | 低（直接读取） | 高（需多次生成或额外模型） |
| 校准改善 | 后处理调整概率 | Prompt工程、Few-shot |
| 适用模型 | 开源模型 | 闭源+开源 |

### 2.4 黑盒UQ方法详细对比

| 方法 | 信息需求 | 采样次数 | AUROC (TriviaQA) | AUROC (CoQA) | 计算成本 |
|------|---------|---------|-----------------|-------------|---------|
| **语义熵** | NLI模型 | m=10 | **0.82** | **0.79** | 高 |
| **Eig(C)** | 仅文本 | m=10 | 0.80 | 0.77 | 中 |
| **Deg(C)** | 仅文本 | m=10 | 0.79 | 0.75 | 中 |
| **Ecc(C)** | 仅文本 | m=10 | 0.78 | 0.74 | 中 |
| Self-Consistency | 仅文本 | m=10 | 0.79 | 0.76 | 中 |
| p(True) | 仅文本 | 1 | 0.74 | 0.71 | 低 |
| Verbalized | 仅文本 | 1 | 0.72 | 0.69 | 低 |
| Mean Token Entropy | logits | 1 | 0.71 | 0.68 | 低 |

> 注：具体数值因模型和数据集而异，上表展示典型趋势。

### 2.5 评估指标

**AUROC（Area Under ROC Curve）**：

$$\text{AUROC} = \frac{1}{|C^+||C^-|} \sum_{c_i \in C^+} \sum_{c_j \in C^-} \mathbb{I}(c_i > c_j)$$

其中 $C^+$ 是正确预测的置信度集合，$C^-$ 是错误预测的置信度集合。

**ECE（Expected Calibration Error）**：

$$\text{ECE} = \sum_{m=1}^M \frac{|B_m|}{n} |\text{acc}(B_m) - \text{conf}(B_m)|$$

其中 $B_m$ 是第 $m$ 个分箱，$n$ 是总样本数。

## 三、黑盒校准的独特挑战

### 3.1 信息受限

```
白盒场景可获取的信息：
├── Token级概率（softmax输出）
├── 隐藏状态（各层表示）
├── 注意力权重
└── 梯度信息

黑盒场景仅能获取：
└── 生成的文本
```

信息受限导致：
- 无法直接计算 Token 级熵
- 无法使用 Temperature Scaling 等后处理方法
- 无法分析模型内部表示

### 3.2 置信度引出的可靠性

| 挑战 | 说明 |
|------|------|
| Prompt敏感性 | 不同Prompt模板引出的置信度差异大 |
| 一致性不足 | 同一问题多次引出可能给出不同置信度 |
| 过度自信 | 模型倾向给出高置信度，即使答案错误 |
| 表达偏差 | 数字置信度 vs 语言置信度的映射不稳定 |

### 3.3 校准改善的局限

白盒场景可用 Temperature Scaling、Platt Scaling 等后处理方法直接校准概率值。黑盒场景只能：

| 方法 | 说明 | 效果 |
|------|------|------|
| Prompt工程 | 优化置信度引出的Prompt | 有限 |
| Few-shot校准示例 | 提供校准良好的示例 | 中等 |
| 外部校准模型 | 训练独立的校准器 | 较好但需额外数据 |

## 四、关键见解与总结

### 4.1 核心结论

1. **黑盒校准是实际需求驱动**：闭源LLM的普及使黑盒场景成为常态
2. **语言化置信度是黑盒核心手段**：虽不完美，但当前最可行的方案
3. **多次采样方法更可靠但更贵**：Self-Consistency、语义熵等效果更好但成本高
4. **校准改善是黑盒最大短板**：缺乏像Temperature Scaling那样简单有效的后处理方法

### 4.2 方法选择指南

| 场景 | 推荐方法 | 原因 | 成本 |
|------|---------|------|------|
| 闭源模型 + 高价值决策 | 语义熵 | 效果最佳 | 高 |
| 闭源模型 + 实时应用 | Verbalized Confidence | 单次生成，速度快 | 低 |
| 闭源模型 + 中等要求 | Eig(C) 或 Deg(C) | 无需额外模型 | 中 |
| 开源模型 | Token熵 + Temperature Scaling | 直接校准概率 | 低 |

### 4.3 未来方向

| 方向 | 说明 |
|------|------|
| 统一置信度引出标准 | 标准化Prompt模板和评估协议 |
| 轻量级黑盒校准 | 降低多次采样的计算成本 |
| 自适应采样策略 | 根据初步结果决定是否需要更多采样 |
| 跨模型校准迁移 | 在一个模型上学习的校准能否迁移到另一个 |
