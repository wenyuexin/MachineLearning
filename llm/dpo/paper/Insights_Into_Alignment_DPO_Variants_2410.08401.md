## 一、核心贡献

本文对当前主流的无强化学习（RL-free）对齐方法进行了**最全面的系统性评估**，主要贡献包括：

1. **三种训练场景的深度对比**
   - 标准流程：SFT → 对齐方法
   - 无SFT：直接在预训练模型上应用对齐
   - 指令模型：使用instruction-tuned模型作为基座

2. **跨13个benchmarks的大规模评估**
   - 覆盖对话系统、推理、数学解题、问答、真实性、多任务理解
   - 包含MT-Bench、BigBench、Open LLM Leaderboard等权威评测

3. **数据规模对性能影响的定量分析**
   - 发现对齐方法在**小数据集（1K-10K）**上即可达到近最优性能
   - 挑战了"对齐需要大量数据"的普遍认知

4. **方法优劣的深度洞察**
   - KTO在大多数任务上表现最优
   - CPO和KTO可在跳过SFT阶段时保持竞争力
   - 对齐方法对复杂推理任务提升有限，但对数学解题和真实性有显著改善

## 二、背景与动机

### 2.1 RLHF的局限性

传统的基于人类反馈的强化学习（RLHF）采用三阶段流程：

```
预训练模型 → SFT → 奖励模型训练 → PPO优化
```

RLHF面临诸多挑战：
- **训练不稳定**：PPO算法本身的复杂性
- **奖励黑客（Reward Hacking）**：模型利用奖励模型的缺陷获得高分
- **计算成本高**：需要同时维护策略模型、价值模型和奖励模型
- **扩展性挑战**：训练过程存在固有的不稳定性和可扩展性问题

### 2.2 相关工作

**RLHF的发展与挑战**

RLHF (Christiano et al., 2023) 通过训练奖励模型，利用Bradley-Terry (BT) 模型通过PPO等强化学习算法优化最大化奖励。虽然RLHF相比单纯的SFT表现出色，但它面临稳定性、奖励黑客和强化学习固有的扩展性等挑战。

**RL-free方法的兴起**

为解决RLHF的问题，研究人员提出了一系列优化相对偏好而不依赖强化学习的方法。

**早期方法：SLiC和RRHF**

- **SLiC (Sequence Likelihood Calibration)** (Zhao et al., 2023)：引入了一种新颖方法来排序由监督微调（SFT）模型产生的偏好，在训练过程中采用校准损失和正则化微调损失。

- **RRHF (Rank Response with Human Feedback)** (Yuan et al., 2023)：利用零边际似然对比损失训练SFT模型，假设每个输入有多个排序响应。

尽管这些方法有效，但SLiC和RRHF缺乏理论基础。

**DPO的理论突破**

DPO提出了一种方法，使用Bradley-Terry (BT)模型直接将SFT模型拟合到人类偏好，为该过程提供了理论见解。

**后续方法：RSO和Zephyr**

- **RSO (Statistical Rejection Sampling Optimization)** (Liu et al., 2024)：结合了SLiC和DPO的方法论，同时引入了通过统计拒绝采样收集偏好对的增强方法。

- **Zephyr** (Tunstall et al., 2023)：利用最先进（SOTA）模型为相同输入生成响应，并使用GPT-4等教师模型进行排名，增强了DPO。他们强调了在使用DPO之前将SFT作为初步步骤的必要性。

**IPO的正则化改进**

IPO (Azar et al., 2023) 类似于DPO方法，在数学上证明了DPO方法在过拟合和泛化方面的局限性，提出了一个从人类偏好学习的综合目标来解决这些问题。

**SPIN的自我训练方法**

SPIN (Self-Play Fine-Tuning) (Chen et al., 2024) 引入了一种自我训练方法来增强DPO，使用SFT步骤中使用的数据集。该方法的关键思想是将合成数据作为拒绝响应，将SFT数据集中的黄金响应作为选择响应。

**KTO的前景理论启发**

KTO (Ethayarajh et al., 2024) 受Kahneman和Tversky关于前景理论的开创性工作启发，旨在最大化LLM生成的效用，而不是最大化偏好的对数似然。这种方法不需要两个相同输入的偏好，因为它专注于判断偏好是可取的还是不可取的。

**CPO的内存效率**

CPO (Constrictive Preference Optimization) (Xu et al., 2024) 提出了一种通过组合最大似然损失和DPO损失函数来高效学习偏好的方法，旨在提高内存和学习效率。研究表明，在训练期间加载参考策略模型(π_ref)是不必要的。通过从内存中省略参考模型，CPO提高了操作效率，使得能够以比DPO更低的成本训练更大的模型。

**研究空白**

尽管这些研究很重要，但缺乏关于对齐方法在完成和偏好学习方面的比较研究。虽然那些研究涉及了无SFT步骤的DPO方法，但进一步探索替代方法是必要的。虽然高质量偏好的重要性被广泛认可，但仍需要探索数据量对对齐方法性能的影响。此外，泛化的关键方面仍未被探索。虽然对齐模型的目标是提高所有类别的性能，但改进对齐方法往往以牺牲其他领域的性能为代价。

**本研究目标**

为了解决这些模糊性，我们：
1. 探索对齐方法的学习能力，旨在缓解DPO框架内的过拟合挑战
2. 在三种不同场景下，检查对齐方法在对话系统、推理、数学解题、问答、真实性和多任务理解方面的有效性
3. 全面评估显示，对齐方法在推理任务上缺乏性能，但在解决数学问题和真实性方面表现出令人印象深刻的表现
4. 观察到在标准对齐过程中，使用较小的训练数据子集用所有对齐算法微调SFT模型可以产生更好的性能（见图3）

### 2.3 RL-free方法对比

为解决RLHF的问题，研究人员提出了一系列**直接偏好优化**方法：

| 方法 | 核心思想 | 关键特点 |
|------|----------|----------|
| **DPO** | 直接优化策略满足Bradley-Terry模型 | 无需显式奖励模型 |
| **IPO** | 修正DPO的过拟合问题 | 引入正则化目标 |
| **KTO** | 从前景理论出发，仅需二元偏好 | 不需要成对偏好数据 |
| **CPO** | 结合NLL损失和偏好损失 | 无需参考模型，内存高效 |

## 三、方法详解

### 3.1 DPO (Direct Preference Optimization)

DPO的核心洞见：偏好数据服从**Bradley-Terry模型**，最优策略有**闭式解**。

**DPO的创新意义**

尽管DPO通过RL-free方法超越了RLHF，但它面临过拟合和需要广泛正则化的约束，这些可能阻碍策略模型的有效性。

**偏好概率模型**（Bradley-Terry）：

$$P(y_w \succ y_l | x) = \sigma(r(x, y_w) - r(x, y_l))$$

**最优策略的闭式解**：

$$\pi^*(y|x) = \frac{1}{Z(x)} \pi_{ref}(y|x) \exp\left(\frac{1}{\beta} r(x,y)\right)$$

由此推导出DPO损失函数：

$$\mathcal{L}_{DPO}(\pi_\theta) = -\mathbb{E}_{(x,y_w,y_l)\sim D} \left[ \log \sigma\left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right]$$

其中：
- $\pi_\theta$: 参数化策略（待优化模型）
- $\pi_{ref}$: 参考策略（通常为SFT模型）
- $\beta$: 控制与参考策略偏离程度的参数
- $\sigma$: sigmoid函数

**DPO的优势**：
- 无需显式奖励模型
- 直接优化策略满足Bradley-Terry模型
- 理论简洁，实现相对简单

**DPO的局限性**：
- 易于过拟合
- 需要广泛的正则化
- 需要同时加载参考模型，内存效率低

### 3.2 IPO (Identity Preference Optimization)

IPO针对DPO的**过拟合问题**进行改进，在数学上证明了DPO方法在过拟合和泛化方面的局限性。

IPO提出了一个DPO的通用形式并重新制定它，以解决过拟合和正则化问题，提出了一个从人类偏好学习的综合目标。

**IPO损失函数**：

$$\mathcal{L}_{IPO}(\pi_\theta) = -\mathbb{E}_{(y_w,y_l,x)\sim D} \left[ h_\pi(y_w, y_l, x) - \frac{\tau^{-1}}{2} \right]^2$$

其中：

$$h_\pi(y_w, y_l, x) = \log \frac{\pi_\theta(y_w|x) \pi_{ref}(y_l|x)}{\pi_\theta(y_l|x) \pi_{ref}(y_w|x)}$$

**IPO的关键改进**：
- 使用平方损失替代对数损失，提供更稳定的优化目标
- 引入正则化参数$\tau$（一个正实数）控制过拟合
- 提出了一个DPO的通用形式，理论更完整
- 更稳定的优化目标，减少了过拟合风险

**IPO的理论贡献**：
- 在数学上证明了DPO的局限性
- 提供了一个综合的学习目标，解决了泛化问题
- 为后续方法提供了理论基础

### 3.3 KTO (Kahneman-Tversky Optimization)

KTO受**前景理论（Prospect Theory）**启发，这是Kahneman和Tversky的开创性工作。核心思想：人类对损失的敏感度高于同等收益。

**KTO的关键创新**：
- **无需成对偏好**：只需判断单个响应是"好"还是"坏"
- **非对称损失**：对"坏"响应的惩罚大于对"好"响应的奖励
- **最大化LLM生成的效用**：而不是最大化偏好的对数似然
- **消除双重偏好需求**：不需要两个相同输入的偏好，专注于判断偏好是可取的还是不可取的

KTO损失函数：

$$\mathcal{L}_{KTO}(\pi_\theta, \pi_{ref}; \beta) = \mathbb{E}_{x,y\sim D} \left[ 1 - ĥ(x,y;\beta) \right]^{\frac{\tau-1}{2}}$$

其中$ĥ(x,y;\beta)$定义为：

$$
ĥ(x,y;\beta) =
\begin{cases}
\sigma\left( \beta \log \frac{\pi_\theta(y|x)}{\pi_{ref}(y|x)} - \mathbb{E}_{x'\sim D}[\beta KL(\pi_\theta \| \pi_{ref})] \right) & \text{if } y \text{ is desirable} \\
\sigma\left( \mathbb{E}_{x'\sim D}[\beta KL(\pi_\theta \| \pi_{ref})] - \beta \log \frac{\pi_\theta(y|x)}{\pi_{ref}(y|x)} \right) & \text{if } y \text{ is undesirable}
\end{cases}
$$

其中：
- $\pi_\theta$: 我们正在优化的模型
- $\pi_{ref}$: 基础参考策略
- $\beta$: 控制与基础参考策略偏离程度的参数
- $\sigma$: logistic函数
- $KL$: 两个分布之间的KL散度
- $\tau$: 正则化参数

**KTO的理论基础**：
- 前景理论：研究表明人类对损失的敏感度高于同等收益
- 在这个算法中，对偏好是否合适或合适与否有清晰理解是至关重要的
- 消除了对替代偏好的必要性

**KTO的实践优势**：
- 数据标注更简单：只需判断"好/坏"，不需要排序
- 数据利用更高效：单条标注数据即可
- 内置KL约束：通过期望KL项隐式控制与参考模型的偏离

### 3.4 CPO (Contrastive Preference Optimization)

CPO旨在**减少显存占用和提高学习效率**，通过消除参考模型加载需求，解决DPO算法中同时加载两个模型导致的低效学习问题。

**CPO的创新动机**：

尽管DPO和IPO增强了DPO模型的性能并解决了DPO的一些缺点，但基于两个偏好的对齐方法有不同的复杂性。为了解决这个问题，KTO研究通过实施仅使用单个偏好的策略来增强DPO方法的有效性。然而，同时加载两个模型导致了DPO算法中的低效学习。为了改进这一点，开发了CPO方法，增强了DPO方法的效率。

**CPO损失函数**：

$$\mathcal{L}_{CPO} = \mathcal{L}_{prefer} + \mathcal{L}_{NLL}$$

其中：

$$\mathcal{L}_{NLL} = -\mathbb{E}_{(x,y_w)\sim D} [\log \pi_\theta(y_w|x)]$$

$$\mathcal{L}_{prefer} = -\mathbb{E}_{(x,y_w,y_l)\sim D} \left[ \log \sigma\left( \beta \log \pi_\theta(y_w|x) - \beta \log \pi_\theta(y_l|x) \right) \right]$$

$$L_{CPO} = L_{prefer} + L_{NLL}$$

其中：
- $\pi_\theta$: 参数化策略
- $y_w$ 和 $y_l$: 表示被选择和被拒绝的响应
- $x$: 是一组源句子
- $\beta$: 参数
- $\sigma$: logistic函数

**CPO的关键优势**：
- **无需加载参考模型**：研究表明，在训练期间加载参考策略模型(π_ref)是不必要的
- **显存占用减少约50%**：通过从内存中省略参考模型，CPO提高了操作效率
- **支持训练更大规模的模型**：能够以比DPO更低的成本训练更大的模型
- **结合最大似然损失**：保持语言能力，避免纯偏好优化可能的语言能力下降

**CPO的理论贡献**：
- 提出了一种高效学习偏好的方法
- 通过组合最大似然损失和DPO损失函数，提高内存和学习效率
- 为资源受限的环境提供了可行的解决方案

### 3.5 方法对比总结

| 方法 | 损失函数类型 | 需要参考模型 | 数据要求 | 主要优势 | 主要局限 | 理论贡献 |
|------|-------------|-------------|----------|----------|----------|----------|
| **DPO** | 偏好对比 | 是 | 成对偏好 | 理论简洁，效果好，直接优化策略满足BT模型 | 易过拟合，需要广泛正则化 | 闭式解，无需显式奖励模型 |
| **IPO** | 正则化偏好 | 是 | 成对偏好 | 缓解过拟合，更稳定的目标 | 超参敏感，理论复杂 | DPO的通用形式，解决泛化问题 |
| **KTO** | 二元偏好 | 是 | 单条标注 | 无需成对数据，标注简单，数据效率高 | 需估计KL项 | 基于前景理论，非对称损失 |
| **CPO** | 偏好+NLL | 否 | 成对偏好 | 内存高效，无需参考模型 | 可能牺牲对齐质量 | 结合最大似然，提高学习效率 |

**方法演进路径**：

```
DPO (理论基础)
    ↓
IPO (正则化改进)
    ↓
KTO (数据效率)
    ↓
CPO (内存效率)
```

**选择指南**：

1. **追求理论简洁** → DPO
2. **担心过拟合** → IPO
3. **数据标注成本高** → KTO
4. **显存受限** → CPO
5. **综合性能最优** → KTO (实验验证)

**内存效率对比**：

- **DPO**: 需要加载策略模型 + 参考模型 = 2个模型
- **IPO**: 需要加载策略模型 + 参考模型 = 2个模型
- **KTO**: 需要加载策略模型 + 参考模型 = 2个模型
- **CPO**: 只需要加载策略模型 = 1个模型 (节省50%显存)

**数据标注成本对比**：

- **DPO**: 需要成对偏好标注（哪个更好）- 相对复杂
- **IPO**: 需要成对偏好标注 - 相对复杂
- **KTO**: 只需要判断"好"或"坏" - 简单
- **CPO**: 需要成对偏好标注 - 相对复杂

## 四、实验设置

### 4.1 三种评估场景

本文设计了三种训练流程来全面评估对齐方法的适用性：

| 场景 | 训练流程 | 核心研究问题 |
|------|----------|--------------|
| **场景1**：标准流程 | 预训练模型 → SFT训练 → 对齐方法 | 标准流程下各方法的相对性能表现 |
| **场景2**：无SFT | 预训练模型 → 对齐方法（跳过SFT） | 对齐方法是否可以不依赖SFT阶段？ |
| **场景3**：指令模型 | 指令模型 → 对齐方法 | 使用已有指令能力的模型作为基座的效果 |

**为什么需要三种场景？**

1. **场景1**是工业界和学术界最常用的标准流程，提供基准对比
2. **场景2**探索降低训练成本的可能性（跳过SFT可减少约50%训练开销）
3. **场景3**验证对齐方法在不同初始化条件下的鲁棒性

- **KTO在GSM8K（数学）上的表现尤为突出**，显示出强大的数学解题能力（见图2中的Mathematics子图）

## 五、关键结果

### 5.1 场景1：标准流程（SFT后对齐）

**MT-Bench评分**（越高越好）：

## 六、技术洞察

### 6.1 为什么KTO表现最好？

1. **非对称损失设计**：受Kahneman和Tversky的前景理论启发，人类对损失的敏感度高于同等收益。KTO利用这一心理学原理，更符合人类对"好/坏"判断的心理模式
2. **单条数据效率**：不需要构造成对偏好，只需判断单个响应是"好"还是"坏"，数据利用更高效
3. **KL约束内置**：通过期望KL项隐式控制与参考模型的偏离，避免了显式正则化的需要
4. **理论优势**：在数学上证明了DPO方法在过拟合和泛化方面的局限性，KTO通过新的损失函数设计解决了这些问题

### 6.2 为什么对齐对推理帮助有限？

对齐方法优化的是**人类主观偏好**，而非**客观正确性**：

```
偏好优化目标: max E[human_preference(y|x)]
推理能力目标: max E[correctness(y|x)]
```

这两者在很多情况下一致，但在复杂推理任务中可能冲突：

- **推理任务需要逻辑推理能力**，这需要更深层的认知能力，超出偏好对齐的范畴
- **对齐方法主要优化响应的"好坏"**，而非逻辑的正确性
- **实验证据**：在所有对齐场景中，推理任务上SFT和其他对齐方法的性能相对可比，表明RL-free算法不会显著影响推理能力
- **对齐与推理的权衡**：改进对齐方法往往以牺牲其他领域的性能为代价，这是泛化的一个关键方面

### 6.3 SFT的作用与局限

**SFT的必要性**：
- 为模型提供基础的指令遵循能力
- 建立初始的响应风格分布
- **在多任务理解上保持优势**：所有实验中，SFT在MMLU上仍优于或与其他对齐方法相当
- 提供高质量的人类示范数据

**SFT的局限**：
- 仅从示范数据学习，无法利用偏好信号
- 高质量示范数据获取成本高
- 需要大量标注数据

**KTO/CPO的突破**：证明了在某些条件下可以跳过SFT阶段

1. **KTO的优势**：
   - 在跳过SFT时，在对话系统上获得与SFT相当的性能
   - 在场景2中，KTO在GSM8K（42.15%）和TruthfulQA（52.98%）上超越SFT
   - 使用较小的训练数据即可达到饱和性能

2. **CPO的内存效率**：
   - 无需加载参考模型，显存占用减少约50%
   - 支持训练更大规模的模型
   - 结合最大似然损失保持语言能力
   - 在跳过SFT阶段时也保持竞争力

3. **DPO和IPO的依赖**：
   - 在跳过SFT时，DPO和IPO在对话系统中表现较差
   - 得分低于SFT基线，表明这些方法严重依赖SFT阶段

### 6.4 数据规模的关键洞察

**小数据集饱和现象**

实验结果显示（见图3）：
- 所有对齐方法在使用较小训练集时表现更好
- 训练集大小在1K到10K之间时达到最优性能
- 超过10K后，性能提升边际收益极小

**理论解释**

在典型对齐过程中，模型对齐的很大一部分发生在SFT阶段。因此，当旨在用KTO、DPO、IPO和CPO等方法增强SFT模型性能时，使用较小的数据集进行训练是有益的。

本质上，使用SFT对齐和用RL-free方法对齐之间存在权衡，以达到最佳性能。

**实践意义**
- 数据质量 > 数据数量
- 对齐阶段可能不需要大量偏好数据
- 资源有限时，1K-10K高质量偏好数据即可

### 6.5 指令模型的独特优势

**真实性的显著提升**
- 使用指令模型作为基座进行对齐，显著影响了真实性
- 在TruthfulQA上，KTO和IPO超越SFT达17.5%
- 基于预训练模型的KTO超越SFT达9.5%

**指令微调的价值**
- SFT基于指令调优已经足够，在推理、数学、问答和多任务理解上与对齐方法性能相当
- 在大多数情况下，SFT比其他对齐方法更优，特别是在需要准备偏好数据集时
- 准备偏好数据集的挑战仍然显著

**性能权衡**
- 虽然对齐方法在某些任务上优于SFT，但挑战在于准备偏好数据集
- 这使得SFT在大多数情况下成为更优选择

### 6.6 与GPT-4的对比发现

**超越GPT-4的领域**
- 在Humanities领域，KTO达到了与GPT-4相同的分数
- 在STEM领域，CPO超越了GPT-4
- 这凸显了对齐方法在与SOTA模型竞争方面的潜力

**与GPT-4的差距**
- 在Coding、Math、Reasoning任务上，对齐方法与GPT-4仍存在巨大差距
- 这表明在需要强逻辑推理的领域，仅靠偏好对齐是不够的

**启示**
- 对齐方法擅长提升模型在人类偏好相关的任务上（如对话、真实性）
- 对于需要强逻辑推理和编程能力的任务，仍需其他技术（如CoT、更多训练数据等）

## 七、实践建议

### 7.1 方法选择决策树

```
是否有成对偏好数据？
├── 否 → 选择KTO（仅需二元标注）
└── 是
    ├── 显存受限？
    │   ├── 是 → 选择CPO（无需参考模型）
    │   └── 否
    │       ├── 担心过拟合？
    │       │   ├── 是 → 选择IPO
    │       │   └── 否 → 选择DPO（最简单）
    │       └── 追求最佳效果？
    │           └── 选择KTO
    └── 追求最佳效果？
        └── 选择KTO
```

### 7.2 资源配置建议

| 资源条件 | 推荐配置 |
|----------|----------|
| **数据极少** (<1K) | KTO + 跳过SFT（使用指令模型作为基座） |
| **数据较少** (1K-10K) | 任何方法均可，优先KTO |
| **数据充足** (>50K) | 标准流程SFT+KTO，或对齐数据降采样 |
| **显存受限** | CPO（无需参考模型） |
| **追求最佳对话效果** | KTO（MT-Bench表现最佳） |

## 八、总结与展望

### 8.1 核心结论

**实验总结**

本文在三种不同场景下（SFT模型、预训练模型、指令模型）评估了DPO、KTO、IPO和CPO等RL-free算法在各种任务（包括推理、数学解题、真实性、问答和多任务理解）上的性能。

**主要发现**

1. **KTO是当前RL-free对齐方法的最优选择**
   - 在所有三个场景中，KTO始终优于其他对齐方法
   - 在MT-Bench上表现最佳
   - 在多数学术benchmark上展现出卓越性能

2. **对齐数据需求可能被高估**
   - 对齐方法对训练数据量特别敏感
   - 在较小数据子集（1K-10K）上表现最佳
   - 使用较小的训练数据子集用所有对齐算法微调SFT模型可以产生更好的性能

3. **KTO和CPO可以跳过SFT阶段**
   - 与DPO不同，KTO和CPO在跳过SFT阶段时在MT-Bench上达到相当性能
   - 为降低对齐成本提供新思路
   - 这对降低对齐成本具有重要意义

4. **对齐方法对推理能力帮助有限**
   - 这些技术在对齐过程中的推理和问答方面没有显著增强模型性能
   - 但在数学解题和真实性方面有显著改进
   - 对齐方法主要优化人类偏好，而非底层推理能力

5. **指令模型对真实性的重要影响**
   - 指令模型显著影响真实性
   - 使用指令模型作为对齐基座，KTO和IPO在TruthfulQA上超越SFT达17.5%
   - 在MT-Bench上，某些对齐方法（如CPO）甚至表现比SFT更差

### 8.2 局限性

**数据准备挑战**
- 准备适当的数据集用于训练对齐方法是一个关键约束
- 高质量偏好数据的获取成本仍然很高

**偏好排序问题**
- 排序多个偏好呈现了另一个可能影响研究质量的局限性
- 如何有效组织和利用偏好数据仍是挑战

**学习与内存效率**
- 学习和内存的低效性也阻碍了对齐研究的进展
- 同时加载两个模型（策略模型和参考模型）导致了DPO算法中的低效学习

**评估成本**
- 使用MT-Bench和AlpacaEval等重要基准进行评估是昂贵的
- 需要访问GPT-4进行评估，增加了研究的成本和复杂性
- 在某些领域，与GPT-4相比仍存在巨大差距

### 8.3 对未来研究的启示

- **数据效率**：探索为什么小数据就能达到饱和，是否存在"关键偏好样本"
- **推理增强**：如何将偏好优化与推理能力提升结合
- **多阶段优化**：KTO+CPO的组合策略是否能有更好效果
- **理论理解**：建立RL-free方法性能的更完整理论框架
- **领域扩展**：本研究专注于对话系统，计划将研究扩展到安全等其他领域

### 8.4 对业界的意义

本文的实证结果为LLM对齐实践提供了重要参考：
- **降低对齐门槛**：小数据+KTO可以在资源受限情况下获得不错效果
- **优化资源分配**：不必过度投入对齐数据收集，可将资源用于其他环节
- **方法选型指南**：根据具体场景（数据类型、显存限制、任务目标）选择合适方法
- **SFT的价值**：在大多数情况下，SFT仍然比其他对齐方法更优，特别是在多任务理解上
- **突破潜力**：某些领域（如Humanities），KTO达到了与GPT-4相同的分数；在STEM领域，CPO甚至超越了GPT-4，这凸显了对齐方法在与SOTA模型竞争方面的潜力
